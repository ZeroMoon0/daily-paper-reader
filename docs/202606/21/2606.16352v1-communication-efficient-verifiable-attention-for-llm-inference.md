## 摘要
远程大语言模型（LLM）服务的计算完整性可能存在问题。对于传统深度神经网络（DNN），现有的TEE防护的DNN分区（TSDP）方法使用可信执行环境（TEE）计算非线性组件，并验证卸载到不可信GPU的线性组件的完整性。然而，将TSDP直接应用于基于Transformer的LLM会带来显著的TEE计算和TEE-GPU通信开销。本文提出了面向LLM推理的高效通信TEE-GPU注意力机制（VeriAttn），用于加速可验证的LLM推理。VeriAttn将注意力的线性和非线性计算都卸载到GPU，而TEE仅执行验证。此外，对于预填充阶段，VeriAttn采用两级流水线来重叠数据移动、TEE前后处理以及GPU计算。对于解码阶段，当键值缓存超过可用GPU内存时，VeriAttn在TEE和GPU之间分区注意力以减少重复的键值传输。在Intel TDX平台上的评估表明，对于6k令牌的提示和10k令牌的输出，在预填充和解码阶段，VeriAttn分别比TSDP加速了2.60-3.38倍和3.86-5.42倍。

## Abstract
Computation integrity of remote large language model (LLM) serving can be questionable. For conventional deep neural networks (DNNs), the existing TEE-shielded DNN partitioning (TSDP) approach uses Trusted Execution Environment (TEE) to compute non-linear components and verify the integrity of linear components offloaded to an untrusted GPU. However, directly applying TSDP to Transformer-based LLMs incurs significant TEE computation and TEE-GPU communication overhead. This paper presents Communication-efficient TEE-GPU Attention (\textsc{VeriAttn}) for accelerating verifiable LLM inference. \textsc{VeriAttn} offloads both linear and non-linear computations of attention to the GPU, while TEE performs verification. Moreover, for prefill, \textsc{VeriAttn} uses a two-level pipeline to overlap data movement, TEE pre-/post-processing, and GPU computation. For decoding, when the key-value cache exceeds available GPU memory, \textsc{VeriAttn} partitions attention across TEE and GPU to reduce repeated key-value transfers. Evaluation on an Intel TDX platform shows that \textsc{VeriAttn} achieves 2.60-3.38$\times$ and 3.86-5.42$\times$ acceleration over TSDP for 6k-token prompts and 10k-token outputs during prefill and decoding, respectively.

## 1. 核心问题与整体含义

- **研究动机**：远程大语言模型（LLM）服务（LLMaaS）存在计算完整性问题——不可信GPU可能篡改中间计算结果或输出。现有两种解决路径：密码学证明（如zkLLM）开销极大；硬件辅助方案（TEE）计算能力弱于GPU。
- **现有方法局限**：TEE防护的DNN分区（TSDP）将非线性操作留在TEE、线性操作卸载到GPU，在传统DNN上效率高，但直接应用于Transformer LLM时产生严重瓶颈：
  - **预填充阶段**：注意力分数和权重矩阵尺寸随序列长度二次增长，导致TEE-GPU间大量传输；Softmax在TEE中计算昂贵。
  - **解码阶段**：KV缓存超出GPU内存后，非驻留KV条目需反复通过TEE-GPU边界移动，通信开销压制计算收益。
  - **串行执行模式**：TEE和GPU交替等待，利用率低下。
- **本文目标**：设计通信高效的TEE-GPU注意力框架，同时保证计算完整性。

## 2. 方法论

### 核心思想
- 将注意力机制的**线性与非线性计算全部卸载到GPU**（包括Softmax的指数运算），TEE**仅执行轻量级验证**，避免TEE计算瓶颈和往返通信。
- 针对预填充和解码分别设计**专用执行流程**：预填充采用两级流水线重叠操作；解码采用TEE-GPU协作处理KV缓存。

### 关键技术细节

#### 线性计算验证：GVFAVeriMM（高斯变体Freivalds算法）
- 利用高斯随机向量投影，检查GPU返回的矩阵乘积与真实乘积的残差是否在容差ε内。
- 预填充：验证 `E V` 的结果，检验 `||E(V w_G) - Ũ w_G||_∞ ≤ ε`，复杂度从O(L²d_h)降至O(L²+L d_h)。
- 解码：增量维护 `V w_G` 投影，单步验证复杂度O(t+d_h)而非O(t d_h)。

#### 非线性计算验证：LPVeriExp（对数乘积验证）
- 验证GPU返回的指数结果 `ŷ_{r,i}`：计算 `log(∏ ŷ_{r,i}^{a_i}) - Σ a_i s_{r,i}` 的残差，用高斯随机加权后与容差ε'比较。
- 定理保证了错误行被接受的伪接受概率可通过增大系数域大小和重复次数降低。
- 数值稳定性：使用`(m, c)`表示法避免乘积上/下溢出。

#### 预填充流水线
- **两级流水线**：
  - 头块级：独立分割注意力头，重叠PRM→SHM拷贝、SHM→GPU传输、GPU计算。
  - 指数行瓦片级：将庞大指数矩阵分割为行瓦片，重叠GPU计算、PCIe传输、TEE验证与行和累积。
- TEE在验证通过后拼接结果并进行值聚合验证和归一化。

#### 解码协作执行
- TEE保留完整KV缓存 `KV_tee`，GPU维护有限驻留缓存 `KV_gpu`。
- 当KV_gpu满时，将非驻留KV划分为两部分：一部分卸载到GPU临时缓冲区（根据卸载比率p），另一部分留在TEE本地计算。
- 两端分别计算未归一化注意力状态 `(U, Z)`，TEE验证GPU返回的指数和值聚合结果后，合并 `o_t = (U_gpu+U_tee)/(Z_gpu+Z_tee)`。
- 卸载比率p通过在线启发式自适应调整。

## 3. 实验设计

- **模型**：4个开源模型——LLaMA3-3B、LLaMA3-8B、Qwen3-14B、Phi4-14B，默认加载为FP16。
- **基准方法**：
  - Full-TEE：整个模型在TEE内运行（无GPU卸载）。
  - TSDP：传统TEE分片方法（线性计算卸载，Softmax留在TEE，串行执行）。
  - zkLLM：基于零知识证明的可验证LLM推理（仅与LLaMA3-8B对比）。
- **评估指标**：

  - 解码：Time-Per-Output-Token（TPOT）平均值。
- **实验场景**：
  - 序列长度：1k–6k token（预填充）；输出长度：2k–10k token（解码，固定500引导prompt，GPU KV容量2k）。
