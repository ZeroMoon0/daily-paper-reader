## 摘要
量化感知训练（QAT）对于极低比特大语言模型（LLM）至关重要。当前的QAT方法主要基于标量量化（SQ），它能够高效优化，但在2比特精度下性能严重下降。另一方面，向量量化（VQ）提供了更高的表示能力，但其离散码本查找阻碍了端到端训练。我们提出LC-QAT，一个2比特权重量化VQ-QAT框架，通过对离散向量进行学习到的仿射映射来表示量化权重，从而产生高质量的PTQ初始化，并实现完全可微的端到端优化，而在训练前向过程中无需显式码本查找。这种强力的训练后初始化使得LC-QAT高度数据高效。跨多种LLM的实验表明，LC-QAT在仅使用0.1%到10%训练数据的情况下，始终优于最先进的QAT方法。我们的结果将LC-QAT确立为极端低比特模型部署的实用且可扩展的解决方案。

## Abstract
Quantization-aware training (QAT) is essential for extremely low-bit large language models (LLMs). Current QAT methods are mainly based on scalar quantization (SQ), which enables efficient optimization but suffers from severe performance degradation at 2-bit precision. On the other hand, vector quantization (VQ) provides substantially higher representational capacity, but its discrete codebook lookup prevents end-to-end training. We propose LC-QAT, a 2-bit weight-only VQ-QAT framework that represents quantized weights via a learned affine mapping over discrete vectors, which yields a high-quality PTQ initialization and enables fully differentiable end-to-end optimization without explicit codebook lookup in the training forward pass. This strong post-training initialization makes LC-QAT highly data-efficient. Experiments across diverse LLMs demonstrate that LC-QAT consistently outperforms state-of-the-art QAT methods while using only 0.1%--10% of the training data. Our results establish LC-QAT as a practical and scalable solution for extreme low-bit model deployment.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型（LLM）参数量巨大，部署在资源受限设备上需要模型量化。在极低比特（如2比特）设定下，量化感知训练（QAT）通常优于训练后量化（PTQ），因为QAT能调整模型参数以补偿量化误差。
- **现有方法的局限**：
  - **标量量化（SQ）**：当前主流QAT方法基于SQ，每个权重独立量化为少数离散值（2比特仅4种取值），表达能力弱，导致严重性能下降，且需要大量训练数据才能恢复。
  - **向量量化（VQ）**：将权重分组后用码本条目表示，表达能力更强（2比特下可表示4^d种码字，d为组大小），但离散码本查找（最近邻搜索）不可微，阻碍端到端训练。现有少数VQ-QAT方法（如PV-Tuning）依赖昂贵的坐标下降或波束搜索，更新低效且不同步。
- **本文目标**：提出一种可微分的端到端VQ-QAT框架，在保持VQ高表达能力的同时，实现标准反向传播训练，并利用高质量PTQ初始化大幅降低数据需求。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
用**线性约束码本**代替传统无约束码本。每个码字通过一个共享的线性仿射映射 `c = A z + B` 生成，其中 `z ∈ {0,1,2,3}^d` 是离散四进制向量，`A∈R^{d×d}`, `B∈R^d` 是可学习的浮点参数。这样，离散索引选择退化为简单的取整和裁剪操作，再经过线性投影，梯度可直接通过线性映射传播，无需显式最近邻搜索和码本查找。

### 关键技术细节
1. **前向传播**：
   - 维护浮点代理权重 `W_p`，通过 `W_z = clip(round(W_p), 0, 2^b-1)` 获取整数权重。
   - 将 `W_z` 按列分组（组大小d），对每个分块 `W_{z,i}` 应用仿射变换：`Ŵ_i^T = A W_{z,i}^T + B 1^T`，得到解码后的量化权重。
   - 前向计算可重写为：`Y = Σ_i (X_i A) W_{z,i}^T + Σ_i (X_i B) 1^T`，其中 `X_i A` 为浮点激活，`W_{z,i}` 为整数矩阵，可复用优化的整数矩阵乘法内核（Int2-FP16 MatMul）。

2. **梯度估计**：
   - 取整操作不可微。采用**可微梯度估计器（DGE）**（Wang et al., 2025）代替直通估计器（STE）。DGE使用光滑近似函数 `f(x) = δ/2 * (1 + sign(2x/δ - 1) * |2x/δ - 1|^{1/k})`，导数 `f'(x) = (1/k) * |2x/δ - 1|^{1/k - 1}`，k控制尖锐度。这避免了STE在最优解附近的不稳定性。

3. **整数权重预处理**：
   - LC-QAT的整数权重 `W_z` 来自先前的PTQ过程，而非从浮点权重量化得到。因此需要反向初始化代理权重 `W_p`：`W_p = s (W_z - t)`，其中 `s=2a/(2^b-1)`, `t=(2^b-1)/2`, `a=√(6/(m+n))`。这使 `W_p` 符合Xavier初始化统计特性（近似零均值、层相关方差），保证训练稳定性。

4. **初始化**：
   - PTQ阶段使用LDLQ算法和Hadamard变换消除离群值，产生初始量化模型。`A` 和 `B` 初始化为 `A = sG`, `B = -sGμ1`，其中G为随机正交矩阵，μ为中心常数，s为缩放因子，确保码本近似零均值、单位方差、弱维度间相关。

### 算法流程
```
输入：原始浮点模型
1. PTQ初始化：通过LDLQ+Hadamard变换获得整数权重W_z和线性映射参数A,B。
2. 预处理代理权重：根据W_z初始化W_p。
3. 对每个训练步骤：
   a. 前向：W_z = clip(round(W_p),0,3) → 分组解码 → 计算输出Y和损失L。
   b. 反向：使用DGE近似梯度 ∂L/∂W_p ≈ ∂L/∂W_z * f'(W_p)，更新W_p。
4. 精调后得到最终量化模型（W_z和A,B）。
```

## 6. 主要结论与发现

1. **LC-QAT在模型能力上显著超越现有VQ-QAT**：在Qwen-3-8B和1.7B上，LC-QAT的平均准确率（72.18%和61.43%）高于PV-Tuning（70.99%和58.05%），且持续提升而不饱和（图4）。
2. **数据效率极高**：仅用4B tokens（约ParetoQ的13%、BitNet的0.1%）即达到或超越当前最优SQ-QAT和VQ-QAT方法。在LLaMA-3-8B上，LC-QAT平均73.42%高于ParetoQ的72.07%。
3. **高质量PTQ初始化是关键**：LC-PTQ初始化损失接近FP16，而SQ初始化偏离严重（图2），这解释了数据效率优势。
4. **端到端训练优于坐标下降**：相同LC-PTQ初始下，LC-QAT比PV-Tuning高4.4个百分点（表5）。
5. **DGE和预处理不可或缺**：去掉DGE导致损失尖峰和训练不稳定；去掉预处理导致损失无法下降（图5）。
6. **推理效率高**：LC-QAT的CUDA内核实现了显著加速（比FP16快1.68倍）。

## 7. 优点

- **方法创新**：线性约束码本将VQ中的离散查找转化为可微分操作，使得VQ-QAT可以端到端训练，是解决VQ不可微难题的有效方案。
- **数据高效**：利用强力的PTQ初始化，将数据需求降低至SOTA QAT方法的0.1%~10%，极大节省训练成本。
- **兼容性好**：前向计算可重写为Int2×FP16 MatMul，可复用已有的SQ优化内核，部署友好。
- **推理加速**：在保持高精度的同时，推理吞吐量优于现有VQ方法（如QuIP#、AQLM）。
- **实验设计平衡**：覆盖多个模型和规模，消融实验充分，对比基线丰富。

## 8. 不足与局限

- **大规模训练的开放性**：论文实验主要基于4B tokens的有限预算，LC-QAT在更大数据量（如30B+）上的可扩展性未验证。论文自身指出此局限性。
- **线性投影的工程代价**：虽然计算复杂度低（O(N n d)），但与高度优化的GEMM内核的交互未系统研究，可能在某些框架下引入开销。
- **校准数据集依赖**：PTQ初始化对校准数据敏感（表9显示不同校准数据导致约2%的准确率差异），可能影响泛化。
- **实验覆盖不足**：
  - 缺少推理链（CoT）基准的评估（如数学推理的逐步骤验证）。
  - 未测试多语言场景或更长上下文。
  - 指令微调仅在一个数据集上进行，通用性待考察。
- **偏差风险**：主要基于英文基准，非英语或非任务导向场景可能存在偏差。论文未讨论社会影响（在Impact Statement中声明无特定社会后果需突出）。
- **参数开销**：d=4时每个权重矩阵增加20个参数，但d=8时增加到72个参数，虽小但需权衡。
