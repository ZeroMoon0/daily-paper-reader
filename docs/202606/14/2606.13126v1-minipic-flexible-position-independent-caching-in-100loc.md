## 摘要
检索增强和智能体工作负载会重复预填充可预测的结构化输入（我们称之为“跨度”），如文档和代码文件。然而，vLLM等引擎中的前缀缓存无法重用KV条目，除非它们与另一个请求共享相同前缀，而生产级推理服务器中的位置无关缓存（PIC）实现通常需要大量的服务器代码修改，或者将KV状态保留在服务器外部，从而产生主机到设备的传输开销。我们提出了极简PIC（MiniPIC）：一种最小、灵活且快速的vLLM设计，由两个要素构成：免位置编码的KV缓存和用户控制的缓存重用原语。MiniPIC在KV缓存中存储未旋转的K向量，在注意力计算中使用每个请求的逻辑位置对K块应用RoPE，并公开三个面向用户且基于令牌的原语：块对齐填充、跨度分隔符（SSep）和提示依赖（PDep），它们修改哈希行为以及有效的块级因果注意力结构。通过少于100行的核心引擎更改加上自定义注意力后端，这些原语足以在同一运行中的vLLM实例内实现多种PIC方法，包括Block-Attention、EPIC和Prompt Cache，同时原生集成KV缓存CPU卸载实现。在2WikiMultihopQA上，采用交错调度的MiniPIC相比基线vLLM将预填充吞吐量提高了49%，将缓存跨度的首令牌生成时间减少了最多两个数量级，保持了未缓存跨度的线性预填充扩展，并且最坏情况下的开销仅为5.7%。

## Abstract
Retrieval-augmented and agentic workloads repeatedly prefill recurring predictable structured inputs (which we call "spans") such as documents and code files. Yet, prefix caching in engines such as vLLM cannot reuse their KV entries unless they share identical prefixes with another request, while Position-Independent Caching (PIC) implementations within production-grade inference servers typically either require substantial server code changes or keep KV state outside the server, incurring host-to-device transfer overhead. We present Minimalistic PIC (MiniPIC): a minimal, flexible and fast vLLM design built from two ingredients: positional-encoding-free KV cache and user-controlled cache-reuse primitives. MiniPIC stores unrotated K vectors in the KV cache, applies RoPE to K tiles inside attention using per-request logical positions, and exposes three user-facing and token-level primitives: block-aligned padding, span separator (SSep), and prompt depend (PDep), that modify hashing behavior and effective block-level causal attention structure. With fewer than 100 lines of core-engine changes plus a custom attention backend, these primitives are sufficient to realize multiple PIC methods, including Block-Attention, EPIC, and Prompt Cache, within the same running vLLM instance, while natively integrating with KV cache CPU offload implementations. On 2WikiMultihopQA, MiniPIC with interleaved scheduling improves prefill throughput by 49% over baseline vLLM, reduces cached-span time-to-first-token by up to two orders of magnitude, preserves the linear prefill scaling of uncached spans, and incurs only 5.7% worst-case overhead.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现代大语言模型（LLM）推理中的长上下文预填充（prefill）开销巨大，尤其在检索增强生成（RAG）和智能体（agentic）工作负载中，系统会重复预填充文档、代码等结构化输入（称为“跨度”或 spans）。然而，现有生产级推理服务器（如 vLLM）的前缀缓存要求请求共享相同前缀才能重用 KV 缓存；而位置无关缓存（PIC）虽允许跨位置重用，但现有实现要么需要大量服务器代码修改（如 SPNL 的 CIDRA 算法），要么将 KV 状态外存引发主机-设备传输开销（如 EPIC、CacheBlend）。  
- **整体含义**：需要一种既高效又易于集成到现有推理引擎（如 vLLM）的 PIC 方案，从而在不增加系统复杂度的前提下显著提升 RAG 等工作负载的预填充吞吐量。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：仅需两个要素即可实现灵活 PIC：  
  1. **无位置编码的 KV 缓存**：存储未旋转的 K 向量（键值），在注意力内核中根据每个请求的逻辑位置实时应用 RoPE 旋转。  
  2. **用户控制的缓存重用原语**：通过三个特殊令牌（来自模型词表）控制块哈希行为和注意力掩码结构。

- **关键技术细节**：  
  - **位置无关 KV 实现**：将未旋转的键向量 $\tilde{k}_j$ 和值向量 $v_j$ 存入 KV 缓存，查询向量正常旋转。注意力内核通过每请求逻辑位置对 K 瓦片进行在线 RoPE 旋转：$\text{score}(t,j) = (R_t q_t)^\top (R_{\pi(j)} \tilde{k}_j)/\sqrt{d}$，其中 $\pi(j)$ 是缓存键在当前请求中的逻辑位置。  
  - **三个用户原语**：  
    - **Padding**：用户将跨度填充到块边界，确保特殊令牌出现在块起始。  
    - **SSep（跨度分隔符）**：出现在块起始时，重置该块的哈希链，使跨度独立缓存，用于 Block-Attention、EPIC 等方法。  
    - **PDep（提示依赖）**：使该块的哈希依赖整个前文，防止后缀块的假命中。  
  - **哈希规则**：如果块以 SSep 开头，则 $h_i = H(h_0, x_i)$；如果以 PDep 开头，则 $h_i = H(h_0, x_{\le i})$；否则 $h_i = H(h_{i-1}, x_i)$。  
  - **交错调度策略 (ISPS)**：将跨度预填充请求与最终提示请求交错提交，避免屏障等待，利用 vLLM 的连续批处理重叠计算。

### 6. 论文的主要结论与发现

- MiniPIC 在 2Wiki 上实现了 49% 的预填充吞吐量提升（vs. baseline vLLM）和 33% 的提升（vs. SPNL）。

### 7. 优点：方法或实验设计上的亮点

- **极简设计**：核心思想明确（位置无关 KV + 用户原语），代码改动极少，易于集成和维护。  
- **灵活性**：三个原语足以表达 Block-Attention、EPIC、Prompt Cache 等多种 PIC 方法，且可扩展至多轮对话、代理框架等。  
- **性能优越**：通过消除复制和旋转开销（SPNL 的 CIDRA 问题），实现更高吞吐量和更低延迟。  
- **原生兼容**：与 vLLM 的 KV 缓存 CPU 卸载无缝集成，无需额外外部服务。  
- **良好工程实践**：遵循模块分解原则，将复杂策略（padding、调度）留在引擎之外。

### 8. 不足与局限

- **实验覆盖不足**：仅使用单一模型（Llama-3-8B 变体）和单一 RAG 数据集，未测试更大模型、其他注意力架构（如 ALiBi、NoPE）或更复杂的 agentic 工作负载。
