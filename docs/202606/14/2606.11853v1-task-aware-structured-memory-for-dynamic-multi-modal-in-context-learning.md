## 摘要
多模态大语言模型依赖上下文学习进行快速任务适应，但其可扩展性受到有限上下文窗口和长多模态序列中键值缓存成本增长的严重限制。现有内存压缩方法通常依赖于刚性令牌删除或样本相关的重要性估计，这引入了偏差，破坏了语义结构（特别是对于视觉表示），并产生了无法适应新查询的静态记忆。我们提出TASM（任务感知结构化记忆），一种无训练框架，通过任务感知、结构保持和动态可访问的内存构建来解决这些限制。TASM采用任务向量引导的压缩，用捕获演示间共享相关性的任务级方向替代样本特定信号。为了保留底层流形，它通过二分图匹配应用语义感知的令牌合并，在不破坏性修剪的情况下聚合令牌。最后，TASM将内存结构化为一个包含紧凑核心记忆和潜在库的层次结构，支持查询自适应的动态检索。评估证实，TASM在高压缩下保持高性能，有效平衡了效率与适应性。

## Abstract
Multi-modal large language models (MLLMs) depend on in-context learning (ICL) for rapid task adaptation, but their scalability is severely limited by finite context windows and the growing cost of key-value (KV) caches in long multi-modal sequences. Existing memory compression approaches typically rely on rigid token removal or sample-dependent importance estimation, which introduces bias, disrupts semantic structure, particularly for visual representations, and yields static memories that cannot adapt to new queries. We introduce TASM (Task-Aware Structured Memory), a training-free framework that addresses these limitations through task-aware, structure-preserving, and dynamically accessible memory construction. TASM employs task-vector guided compression to replace sample-specific signals with a task-level direction that captures shared relevance across demonstrations. To preserve the underlying manifold, it applies semantics-aware token merging via bipartite graph matching, aggregating tokens without destructive pruning. Finally, TASM structures memory into a hierarchy comprising a compact Core Memory and a Latent Bank, facilitating query-adaptive dynamic retrieval. Evaluations confirm TASM maintains high performance under heavy compression, effectively balancing efficiency with adaptability.

## 1. 核心问题与整体含义

**研究动机与背景**  
多模态大语言模型（MLLMs）依赖上下文学习（ICL）进行快速任务适应，但随着演示样本数量增加（Many-Shot场景），有限上下文窗口和长序列中键值（KV）缓存的成本急剧膨胀，成为主要性能瓶颈。现有记忆压缩方法（如基于注意力得分的硬剪枝）存在三个结构性缺陷：  
- **样本特定偏差**：依赖单次演示的注意力分数，可能丢弃对测试查询关键但对该演示不重要的上下文。  
- **拓扑破坏**：硬剪枝（如Top-K选择）会破坏视觉令牌固有的2D空间流形，影响空间定位等任务。  
- **静态记忆刚性**：压缩后的缓存固定不变，无法在遇到复杂查询时恢复被丢弃的细粒度信息。

**整体含义**  
本文提出一种无需训练的框架**TASM**，通过任务感知、结构保持和动态可访问的记忆构建来克服上述限制，使得多模态上下文学习在保持高性能的同时大幅降低内存占用，支持在消费级硬件上运行长上下文推理。

## 2. 方法论

**核心思想**  
将KV缓存视为对任务的**结构化表示**，而非静态缓冲区。压缩引导方向从“样本特定信号”转为“任务级方向”，并采用软合并代替硬剪枝，配合层次化记忆架构实现查询自适应的动态检索。

**关键技术细节**  

- **任务向量引导压缩**  
  利用上下文中的问答对提取一个**全局任务向量τ**，代表从问题到答案的变换方向。通过将每个令牌的键表示投影到τ上计算重要性分数，并结合层自适应门控（浅层侧重局部注意力，深层侧重任务向量），从而过滤噪声、保留与任务正相关的语义信息。

- **语义感知令牌合并**  
  将令牌分为**Sink节点**（高分保留）和**Source节点**（低分待合并），在二分图上求解最优分配矩阵。对视觉令牌额外施加**空间约束**（局部窗口内才允许合并），保证合并后的令牌仍保留原始图像的局部特征几何结构，避免破坏空间拓扑。

- **查询自适应动态检索**  
  构建层次记忆：**核心记忆**（Core Memory，驻留GPU）和**潜在库**（Latent Bank，可卸载至CPU）。推理时，通过监测注意力分布的**詹森-香农散度**（JS Divergence）来判断当前查询是否需要从潜在库中获取额外令牌。仅在必要时触发稀疏检索，从而实现平均O(1)的复杂度。

**算法流程（文字说明）**  
1. 离线阶段：使用少量演示样本计算任务向量τ；对待压缩的完整上下文，先计算每个令牌的重要性并分层门控融合；  
2. 预填充阶段：为每个注意力层执行语义感知令牌合并，将保留的合并令牌存入核心记忆，其余令牌移入潜在库；  
3. 在线生成阶段：每步解码时，根据JS散度决定是否从潜在库检索Top-k相关令牌，动态扩展当前缓存后执行标准注意力。

1. **高压缩、高性能**：TASM可将上下文长度减少**78.6%**（如ImageNet-100从16264降至3485），同时在所有9个基准上**匹配甚至超越**完整上下文的MLoC性能，并显著优于EMLoC。

3. **动态检索克服静态刚性**：在长视频（VideoMME长视频段）和V-NIAH任务中，TASM通过JS散度触发检索，恢复全上下文性能，而EMLoC因静态丢弃而严重下降。  
4. **任务向量有效且可迁移**：仅需4个演示样本即可提取稳定的任务向量，且可在不相干类别间零样本迁移（仅降0.8%），说明其捕捉的是任务级抽象方向而非样本过拟合。  
5. **效率可扩展**：在32k上下文下，TASM保持11.5GB显存和43.8 tok/s解码速度，而完整上下文已OOM。

## 8. 不足与局限

- **模型规模局限**：仅测试7B/8B模型，未在更大模型（如13B/34B）上验证，大模型的行为可能有差异。
