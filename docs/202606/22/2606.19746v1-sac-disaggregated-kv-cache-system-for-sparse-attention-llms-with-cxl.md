## 摘要
大型语言模型向长上下文推理的扩展已将主要服务系统瓶颈从计算转向内存容量。传统的密集注意力模型解决方案依赖基于RDMA的解聚内存池，在解码前从远程存储粗粒度地获取整个前缀KV缓存到本地内存。然而，对于新兴的稀疏注意力模型，这种方法根本上是低效的。尽管解码过程中只有一小部分KV条目是活跃的，但这些系统仍然在本地获取完整的KV缓存，导致严重的传输瓶颈和本地内存浪费。为了解决这个问题，我们提出了SAC，首个针对稀疏注意力模型优化的高效解聚KV缓存系统。通过利用Compute Express Link（CXL）的低延迟、缓存行粒度加载/存储语义，SAC在推理过程中仅按需获取所需的top-k KV条目。在DeepSeek-V3.2上使用SGLang的评估表明，与基于RDMA的基线相比，SAC实现了2.1倍的吞吐量提升、9.7倍的TTFT降低和1.8倍的TBT降低，确立了基于CXL的解聚作为新兴稀疏注意力模型的优越基础设施。

## Abstract
The scaling of LLMs toward long-context inference has shifted the primary serving system bottleneck from computation to memory capacity. Traditional solutions for dense attention models rely on RDMA-based disaggregated memory pools, which perform coarse-grained fetching of the entire prefix KV cache from remote storage to local memory before decoding. However, this approach is fundamentally inefficient for emerging sparse attention models. While only a small fraction of KV entries are active during decoding, these systems still fetch the full KV cache locally, leading to severe transmission bottlenecks and local memory wastage. To address this, we propose SAC, the first efficient disaggregated KV cache system optimized for sparse attention models. By leveraging the low-latency, cache-line granularity load/store semantics of Compute Express Link (CXL), SAC fetches only the required top-k KV entries on demand during inference. Evaluations on DeepSeek-V3.2 using SGLang show that SAC achieves 2.1x higher throughput, 9.7x lower TTFT, and 1.8x lower TBT compared to RDMA-based baselines, establishing CXL-based disaggregation as the superior infrastructure for emerging sparse attention models.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：长上下文LLM推理中，KV缓存容量需求激增，计算不再是主要瓶颈，内存容量成为关键限制。传统解决方案采用基于RDMA的分离式内存池，在解码前将整个前缀KV缓存预取到本地内存。
- **核心问题**：对于新兴的**稀疏注意力模型**（如DeepSeek-V3.2、GLM-5.1），只有一小部分KV条目（top-k）在解码中活跃。RDMA的全量预取策略导致：
  - **传输瓶颈**：预取大量非必需KV数据占用RDMA带宽，造成长队列延迟，损害首Token时间（TTFT）和整体吞吐量。
  - **本地内存浪费**：仅21%的KV缓存（128K上下文）被实际使用，却必须驻留本地，迫使系统配备TB级本地内存，成本高昂且效率低下。
- **整体含义**：迫切需要一种能按需、细粒度、低延迟获取活跃KV条目的分离式KV缓存架构，以匹配稀疏注意力模型的访问模式。

## 2. 方法论

- **核心思想**：利用CXL（Compute Express Link）的低延迟、缓存行粒度load/store语义，在解码过程中**仅按需获取每个注意力层所需的top-k KV条目**，而非预取全部数据。CXL将KV缓存存储于全局共享的CXL内存池，各计算实例直接通过内存访问指令读取。
- **系统设计（SAC）**：
  - **Prefill Instance**：在预填充阶段，计算完整KV缓存并写入CXL内存池（写入采用向量化存储内核，利用 `st.global.b64` 指令高带宽写入）。
  - **Decode Instance**：基于SGLang的HiSparse框架，每个解码层由Top-K选择器确定当前query所需的KV索引，然后通过**优化的读取内核**（向量化加载 `ld.global.b64`）从CXL内存按需获取稀疏的top-k条目到GPU HBM。新生成Token的KV也写回CXL内存。
  - **CXL管理**：
    - 统一地址空间，元数据也托管在CXL共享内存中，避免RPC开销。
    - 采用**层优先（layer-first）** 的内存布局，与本地GPU内存布局一致，简化访问逻辑。
    - **设备交织（Device Interleaving）**：将不同请求的KV缓存分配到不同CXL设备，通过轮询调度使GPU访问不同设备，缓解单链路压力，提升带宽利用率（图13显示吞吐量提升9.2%）。
- **关键技术细节**：
  - CXL设备初始化为DAX设备，通过 `mmap` 映射到主机虚拟地址，再用 `cudaHostRegister` 使GPU可直接访问。
  - 读取路径适配稀疏、非连续访问模式，每次数据传输量仅为top-k（2048个）向量。

## 6. 主要结论与发现

- SAC在端到端解码性能上：
  - 相比RDMA基线，**吞吐量提升2.1倍**，TTFT降低9.7倍，TBT降低1.8倍。
  - 相比本地DRAM非分离式上界，吞吐量仅损失9%（91%的性能），TTFT和TBT接近（仅有微小增加）。
- 可扩展性：随着并发度增加，RDMA吞吐量快速饱和（受传输带宽限制），而SAC持续线性增长，在128K上下文下最高获得**3.1倍**于RDMA的吞吐量。
- CXL的细粒度按需获取有效消除了传输瓶颈和本地内存浪费，使系统能支撑更高并发，无需TB级本地内存。
- 设备交织可缓解CXL链路争用，获得9.2%额外吞吐量提升；增大GPU缓冲区（6K vs 4K）降低CXL访问次数，吞吐量提升10.4%。
- 结论：CXL分离式KV缓存是稀疏注意力模型服务的高效基础设施，优于RDMA方案。
