## 摘要
高上下文代理给键值（KV）缓存带来了异常压力：长前缀在多个短轮次中被重复使用，而并发性决定了服务系统能否保持GPU利用率。我们针对这一场景研究了4位KV缓存压缩，采用TurboQuant风格的旋转和码本量化作为质量锚点，并以vLLM FP8 KV缓存作为部署锚点。我们报告了三项贡献。首先，我们将4位KV缓存围绕多轮代理工作负载进行构建，其中任务质量、缓存驻留和服务吞吐量必须联合衡量。其次，我们描述了使4位路径鲁棒所需的设计选择，包括非对称K/V处理、Walsh-Hadamard旋转、QJL移除以及块级缩放变体。第三，我们展示了在AMD GPU上的服务优化，包括优化的解码注意力内核和UltraQuant——一种FP4近似路径，该路径使用FP8查询、FP4 KV张量、UE8M0组缩放以及CDNA4上的原生缩放MFMA支持。在一个长上下文、多轮的代理工作负载上，UltraQuant在缓存压力大的后期轮次中将P50首次令牌延迟降低了3.47倍（所有轮次平均降低2.3倍），并将输出吞吐量提升至FP8 KV基线的1.63倍。

## Abstract
Context-heavy agents place unusual pressure on the key-value (KV) cache: long prefixes are reused across many short turns, while concurrency determines whether the serving system can keep GPUs utilized. We study 4-bit KV-cache compression for this setting, using TurboQuant-style rotation and codebook quantization as a quality anchor and vLLM FP8 KV caching as the deployment anchor. We report three contributions. First, we frame 4-bit KV caching around multi-round agent workloads where task quality, cache residency, and serving throughput must be measured jointly. Second, we describe the practical design choices needed to make the 4-bit path robust, including asymmetric K/V treatment, Walsh-Hadamard rotation, QJL removal, and block-scale variants. Third, we present serving optimizations on AMD GPUs, including optimized decode-attention kernels and UltraQuant, an FP4 approximation path that uses FP8 queries, FP4 KV tensors, UE8M0 group scales, and native scaled-MFMA support on CDNA4. On a long-context, multi-turn agentic workload, UltraQuant cuts P50 time-to-first-token by 3.47x in the cache-pressured late rounds (2.3x across all rounds) and raises output throughput by 1.63x over the FP8 KV baseline.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- 采用4位量化，结合旋转（rotation）和码本（codebook）思想，但为了硬件效率，最终用FP4微张量配合组缩放（UE8M0）替代码本，使得反量化（dequantization）融入到CDNA4矩阵核指令中，避免软件查表开销。
- 提出两个端点：**Ultra-TQ**（保留TurboQuant码本表示，但通过内核优化部署）和**UltraQuant**（采用FP4+UE8M0组缩放，由矩阵核原生处理）。

### 2.2 关键技术细节
- **Walsh-Hadamard旋转**：将原始KV缓存分布旋转为近高斯分布，使4位标量量化更有效。
- **非对称K/V处理**：键（K）和值（V）分别处理，因为K误差影响softmax分布，V误差影响注意力输出。
- **校准质心（Calibrated Centroids）**：对部分层（约10%）用单次前向传播校准码本质心，降低量化MSE约10.3%。
- **块级缩放**：每32个通道一组，用组内的绝对最大值（absmax）和全局常数c进行缩放，映射到FP4 E2M1网格上，缩放因子为UE8M0（8位无符号指数）。全局常数c通过最小化重建误差离线优化得到（c=0.156），模型无关。
- **FP4路径**：KV缓存存储为每个组32个FP4码（每字节2个码）加1个UE8M0缩放字节，有效每元素4.25比特。反量化时，值 = code × 2^(byte-127)，直接由CDNA4的缩放MFMA指令（MFMA_SCALE_F32_*_F8F6F4）吸收，无需软件反量化。
- **查询（Q）预处理**：将查询也做Hadamard旋转并舍入到FP8，然后与FP4键进行缩放MFMA。
- **边界层保护**：保留第一层和最后两层注意力层的BF16 KV缓存（每侧n=2）。

### 2.3 公式/算法流程（文字说明）
- UltraQuant编码：对K/V做旋转 → 分32通道组 → 每组计算absmax m → 计算UE8M0指数E = round(log2(c·m)) → 每个坐标量化到最近FP4等级 → 打包存储。
- 解码：查询旋转并转FP8 → 加载码和UE8M0字节 → 通过缩放MFMA计算QK和PV。
