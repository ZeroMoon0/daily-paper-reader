## 摘要
扩散大语言模型（dLLM）在每个去噪步骤中重新编码整个前缀，导致重新计算的计算量与上下文长度呈二次方关系，使得长上下文场景下计算代价过高。我们提出Prefilling-dLLM，一种无需训练的预填充-解码分离框架，将前缀划分为N个块，一次性缓存其KV表示，并在解码时利用块内令牌稀疏性选择最相关的top-K个块，证明稀疏预填充在性能上可超越密集注意力，同时将每步复杂度从全序列长度的二次方降低到仅解码长度的二次方。在LongBench和InfiniteBench上，Prefilling-dLLM在dLLM加速方法中达到了最先进的质量，而一个将解码并行化到非连续缓存块KV上的注意力内核在8K-32K上下文中实现了9.1-28.0倍的加速。我们进一步展示，附加在每个块开头的序列起始令牌充当周期性注意力锚点，消除了“中间丢失”现象。代码见https://github.com/menik1126/Prefilling-dLLM。

## Abstract
Diffusion large language models (dLLMs) re-encode the entire prefix at every denoising step, causing recomputation that scales   quadratically with context length and becomes prohibitive for long-context scenarios. We propose Prefilling-dLLM, a training-free   prefill-decode disaggregation framework for dLLMs that partitions the prefix into N chunks, caches their KV representations once,   and selects the top-K most relevant chunks with intra-chunk token sparsity for decoding, showing that sparse prefilling can   outperform dense attention while reducing per-step complexity from quadratic in the full sequence length to quadratic only in the   decode length. On LongBench and InfiniteBench, Prefilling-dLLM achieves state-of-the-art quality among dLLM acceleration methods,   and an attention kernel that parallelizes decoding over the non-contiguously cached chunk KV yields 9.1--28.0x speedup at 8K--32K   contexts. We further show that beginning-of-sequence tokens prepended to each chunk act as periodic attention anchors that eliminate   the lost-in-the-middle phenomenon. Code is available at https://github.com/menik1126/Prefilling-dLLM.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：扩散大语言模型（dLLM）在长上下文推理中，每个去噪步骤都需要对整个前缀（prefix）进行重新编码，导致计算复杂度与上下文长度呈二次方增长（O((L_p+L_d)^2·T)），使得长上下文场景下计算开销不可接受。
- **背景**：与自回归模型不同，dLLM 在每一步都进行双向注意力，无法像 AR 那样自然地累积 KV 缓存。现有加速方法（如 Fast-dLLM、Sparse-dLLM、dKV-Cache）均在标准推理循环内操作，未避免重复的全前缀计算。自回归领域的预填充-解码分离架构尚未被引入 dLLM。
- **整体含义**：论文旨在通过系统级分离前缀计算与迭代解码，消除冗余的前缀重新编码，从而大幅加速长上下文 dLLM 推理，同时保持甚至提升生成质量。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
将前缀划分为 N 个固定大小的块，在**单独预填充阶段**一次性计算并缓存每个块的 KV 表示（仅块内自注意力），然后通过**预测性评分**选出 top-K 最相关的块，在解码阶段只使用这些块的缓存 KV 进行稀疏交叉注意力，同时利用**块内令牌稀疏性**进一步压缩每个块内的令牌数量。

### 关键技术细节
1. **分块预填充（Chunk Prefill）**：
   - 前缀分割为 N = ⌈L_p/C⌉ 个块，每个块前附加一个 BOS 令牌作为分割符。
   - 对每个块独立进行块内自注意力，得到 KV 缓存 (K_i, V_i)。
   - 预填充复杂度 O(N·C²)，且可跨设备并行。

2. **预测性评分（Predictive Score）**：
   - **自信息得分** s_I(c_i)：查询窗口 q 在给定块下的负对数似然。
   - **伪标签得分** s_P(c_i)：伪标签 m（由初始粗略扩散生成）在给定块和查询下的负对数似然。
   - 最终得分 s(c_i) = s_I(c_i) + s_P(c_i)，选出 top-K。

3. **双层稀疏注意力**：
   - **块间稀疏**：仅保留 top-K 个块（K ≪ N），解码时响应令牌只与这些块做交叉注意力。
   - **块内稀疏**：在每个选中的块内，根据查询与块令牌的**双向注意力权重**，只保留得分最高的 top-B 个令牌。
   - 解码每步复杂度：O(L_d² + K·B·C)，其中 L_d 为解码长度，B 为每块保留令牌数。

4. **解码前缀复用**：选中的块 KV 缓存固定不变，跨所有去噪步骤共享；查询和响应令牌每步重新计算，但计算量仅与解码长度相关。

### 流程示意（文字）
- **预填充阶段**：分块 → 并行计算各块 KV 缓存 → 用伪标签和查询生成伪标签 → 计算自信息得分 + 伪标签得分 → 排序选出 top-K → 对每个 top-K 块再进行令牌级重要性排序 → 保留 top-B 令牌。
- **解码阶段**：固定缓存 top-K 块的稀疏 KV → 每步迭代去噪，响应令牌与缓存 KV 做交叉注意力 → 逐步解开掩码。

## 6. 论文的主要结论与发现

1. **Prefilling-dLLM 显著加速长上下文推理**：在 8K/16K/32K 上下文下分别获得 9.1×/16.1×/28.0× 的端到端加速，吞吐量优于所有加速方法（包括 Sparse-dLLM）。
2. **稀疏预填充优于密集注意力**：在 LongBench 和 InfiniteBench 上，Prefilling-dLLM 在大多数子任务上取得最好或次好结果，尤其检索密集型任务（如 RB-P、Passkey、Number Retrieval）。
3. **块内稀疏性进一步改善性能**：加入块内令牌选择性保留后，平均得分比仅块间稀疏更高（Dream: 34.59 vs 22.01）。
4. **消除“丢失在中间”现象**：通过块级 BOS 令牌作为周期性注意力锚点，Prefilling-dLLM 在不同上下文位置均保持高检索准确率（>90% EM），而 Vanilla 模型在 16K/32K 时显著下降。
5. **注意力锚点非注意力 sink**：Chunk BOS 令牌仅吸收 0.30% 平均注意力，远低于 AR 模型中的 20–60%，不会成为注意力病灶。
6. **多块预填充优于单块**：固定总预算（4096 令牌）下，较小的块（256/512）在 128K 超长上下文下取得 92.3% EM，而单块（4096）降至 68%。

## 8. 不足与局限

1. **静态块选择**：top-K 块在预填充后固定，解码过程中不能动态调整，若伪标签不准确，可能遗漏关键上下文。
