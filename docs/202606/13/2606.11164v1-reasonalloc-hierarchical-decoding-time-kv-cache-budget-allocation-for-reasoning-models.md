## 摘要
大型语言模型（LLM）推理中的长思维链（CoT）轨迹因键值（KV）缓存快速增长而导致严重的推理瓶颈。当前的解码时压缩方法通过标记驱逐来缓解这一问题，但通常假设所有层和头部的预算分配均匀。相比之下，现有的非均匀预算分配方法主要设计用于静态提示预填充阶段，并且不能捕捉自回归推理的逐步上下文需求。为弥补这一差距，我们提出ReasonAlloc，一个无训练框架，将解码时KV压缩重新定义为层次化预算分配问题。ReasonAlloc在两个互补层面运作：离线层预分配策略捕捉一种架构驱动的需求模式，我们称之为“推理波”；而在线头部策略则根据实时效用，在解码过程中将资源重新分配给信息丰富的头部。在数学推理基准（MATH-500、AIME 2024）上使用DeepSeek-R1-Distill-Llama-8B、DeepSeek-R1-Distill-Qwen-14B和AceReason-14B的评估显示，ReasonAlloc优于均匀预算的R-KV、SnapKV和Pyramid-RKV（一种强制执行静态单调递减层预算的基线），在较小预算（128-512个标记）下增益最大。ReasonAlloc可与现有标记驱逐策略即插即用，并引入可忽略的推理时开销。

## Abstract
Long chain-of-thought (CoT) trajectories in large language model (LLM) reasoning cause severe inference bottlenecks due to rapid key-value (KV) cache growth. Current decoding-time compression methods mitigate this issue via token eviction, but typically assume a uniform budget distribution across all layers and heads. In contrast, existing non-uniform budget allocation methods are predominantly designed for the static prompt prefill phase, and they do not capture the stepwise context demands of autoregressive reasoning. To bridge this gap, we propose ReasonAlloc, a training-free framework that recasts decoding-time KV compression as a hierarchical budget allocation problem. ReasonAlloc operates at two complementary levels: an offline layer-wise preallocation strategy captures an architecture-driven demand pattern which we call ``\textit{Reasoning Wave}'', while an online head-wise strategy reallocates resources during decoding to information-rich heads based on real-time utility. Evaluations on mathematical reasoning benchmarks (MATH-500, AIME~2024) using DeepSeek-R1-Distill-Llama-8B, DeepSeek-R1-Distill-Qwen-14B, and AceReason-14B show that ReasonAlloc outperforms uniform-budget R-KV, SnapKV, and Pyramid-RKV (a baseline enforcing a static, monotonically decreasing layer budget), with the largest gains at small budgets (128-512 tokens). ReasonAlloc is plug-and-play with existing token-eviction policies and introduces negligible inference-time overhead.

## 1. 核心问题与整体含义（研究动机和背景）
- **动机**：大型推理模型（如 DeepSeek-R1 系列）在长思维链推理时生成很长的 token 序列，导致 KV 缓存不断增长，成为严重的内存瓶颈，限制批处理大小和推理吞吐量。
- **现有方法不足**：当前的解码时 KV 压缩方法（如 R-KV、SnapKV）通常假设各层和各头的缓存预算均匀分配；而现有的非均匀分配方法（如 PyramidKV）主要针对静态的 Prompt 预填充阶段设计，缺乏对自回归解码过程中逐步变化的上下文需求的自适应能力。
- **本文目标**：提出一个解码时的层次化预算分配框架，将 KV 压缩视为资源路由问题，在层和头两个级别上非均匀、动态地分配预算，从而在低预算下保持推理精度。

## 2. 论文提出的方法论
### 核心思想
- 将解码时 KV 压缩分解为两个互补的层次：
  1. **离线层预分配（Offline Layer-wise Preallocation）**：通过对少量随机提示进行轻量校准，捕捉由架构决定的层间需求模式（称为“推理波”），生成稳定的层预算。
  2. **在线头动态路由（Online Head-wise Dynamic Routing）**：在解码过程中每隔 Δ 个 token 刷新，基于实时重要性-冗余评分动态将层预算重新分配给各个注意力头。

### 关键技术细节
- **层预分配**：
  - 对每层 ℓ，计算维持目标注意力质量 ρ（取 0.93）所需的最小 token 数（原始需求 B_raw^(ℓ)）。
  - 通过“鲁棒化操作符”处理原始需求向量：幂平滑（指数 γ=0.5）→ 全局缩放 → 裁剪到 [0.25*平均预算, 2*平均预算] → 再次归一化，确保总和等于全局预算 B，并防止极端分配。
- **头动态路由**：
  - 每隔 Δ=128 步，对层 ℓ 内所有头的 token 评分 S_j,t 进行全局排名，取第 B^(ℓ) 大的值作为阈值 τ。
  - 统计每个头 i 内评分超过 τ 的 token 数量作为原始需求 r̂_i。
  - 同样应用鲁棒化操作符（指数 β=0.5，最小保护下限 μ=0.25*平均头预算），得到最终头预算 B^(ℓ,i)。
- **通用 token 评分**：框架可兼容任意 token 驱逐策略，本文实例化为 R-KV 的联合重要性-冗余评分：S = α*I + (1-α)*R，其中 α=0.1。
- **算法流程**：参见论文 Algorithm 1，分两个阶段：离线校准生成层预算；解码过程中每 Δ 步计算头预算并执行 token 驱逐。

## 6. 论文的主要结论与发现
- 推理模型的 KV 缓存需求在层和头上高度不均匀，呈现“推理波”模式（浅层高需求、中层低振荡、深层尖峰），且主要由架构决定，任务影响较小。
- 现有均匀分配或静态非均匀分配（如 Pyramid-RKV）在低预算下性能损失严重；ReasonAlloc 通过层次化分配显著提升低预算的准确率（例如 MATH-500 在 512 预算下达到 82.50%，高于 R-KV 的 76.48%）。
- 结合离线层预分配和在线头动态路由可获得最佳性能；单独头路由在小预算下主导增益，单独层路由在中高预算下也能改善。
- 方法引入极低的推理开销，可实现高达 5.52× 的端到端加速。
