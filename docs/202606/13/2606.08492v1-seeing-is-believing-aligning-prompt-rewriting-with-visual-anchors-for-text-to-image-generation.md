## 摘要
尽管文本到图像（T2I）模型具有令人印象深刻的能力，但由于用户提示的简洁性和模糊性，意图生成差距常常存在。现有方法主要完善提示的流畅性和可读性。然而，增强过程仍然缺乏视觉基础。因此，重写器可能会过度推断缺失的细节，导致意图生成差距。为了解决这一限制，我们提出了FaithRewriter，一种用于T2I生成的新型提示增强框架。具体来说，FaithRewriter首先利用多模态MLLM从原始提示生成图像作为中间视觉线索。然后将该线索与提示结合，输入到大规模LLM中，以生成具有视觉基础的增强内容，更好地反映预期内容在图像中的呈现方式。最后，将这些增强内容蒸馏到小规模LLM中以便高效部署，增强其生成有效T2I提示的能力。实验表明，FaithRewriter生成的提示比强基线更忠实于用户意图，视觉上更合理，有助于缩小意图生成差距。

## Abstract
Despite the impressive capabilities of text-to-image (T2I) models, an intent-generation gap often persists due to the brevity and ambiguity of user prompts. Existing approaches primarily polish the prompt for fluency and readability. However, the enhancement process still lacks visual grounding. As a result, the rewriter may over-infer missing details, causing an intent-generation gap. To address this limitation, we propose FaithRewriter, a novel prompt-enhancement framework for T2I generation. Specifically, FaithRewriter first leverages a multimodal MLLM to generate an image from the original prompt as an intermediate visual cue. This cue is then combined with the prompt and fed into a large-scale LLM to produce visually grounded augmentations that better reflect how the intended content should appear in images. Finally, these augmentations are distilled into a small-scale LLM for efficient deployment, enhancing its ability to generate effective T2I prompts. Experiments show that FaithRewriter yields prompts that are more faithful to the user intent and more visually plausible than strong baselines, helping narrow the intent-generation gap.

- **核心问题**：文本到图像（T2I）生成中，用户输入的提示词往往简短、模糊，导致模型生成的结果与实际意图之间存在“意图-生成差距”（intent–generation gap）。
- **现有方法的不足**：现有的提示重写方法（如 LLM 扩展）仅优化语言流畅性和可读性，但**缺乏视觉基础**。重写器在纯文本空间中操作，容易过度推断缺失细节，产生语意通顺但视觉上不可实现（如违反物理定律、错误绑定属性）的“文本幻觉”。
- **核心洞察**：最有效的校验方法是“实际画出来”——通过生成中间图像（视觉锚点）来暴露隐藏的视觉错误，然后据此修正提示。

## 2. 论文提出的方法论

### 2.1 核心思想：“Simulate-Then-Distill”（先模拟再蒸馏）

- 离线阶段：利用多模态大模型（MLLM）生成视觉锚点（Visual Anchor），进行诊断、修正、循环验证，构造高质量偏好数据对（好提示 vs 坏提示）。
- 线上推理阶段：将上述知识蒸馏到小型纯文本 LLM 中，使其无需调用 T2I 模型或 MLLM 即可生成视觉可靠的提示。

### 2.2 关键技术细节

1. **视觉锚点（Visual Anchor）**：将纯文本重写结果 T1 渲染为图像 I1，将隐式的文本失败转化为可视的物理/空间错误。
2. **结构化诊断与修正**：MLLM 检查 I1 是否违反原始意图 T0，识别六种常见失败类型（空间逻辑、属性绑定、物理动力学、计数与几何、否定与缺失、隐式概念），并基于视觉证据生成修正提示 T2。
3. **循环验证（Cycle Verification）**：将 T2 再次渲染得到 I2，由 MLLM 判断三点：
   - 是否修复了 I1 中的问题（Fix Verification）
   - 是否保留了 T0 的所有意图（Intent Preservation）
   - 是否引入新错误（Anti-Regression）
   - 仅通过三者才保留偏好对（Twin = T2, Tlose = T1）。
4. **偏好蒸馏**：先对正样本对 (T0, Twin) 进行监督微调（SFT），再使用直接偏好优化（DPO）在硬负样本对（Twin > Tlose）上训练。DPO 损失函数为：
   ```
   L_DPO(θ) = -E_{(T0, T_win, T_lose)∼D} [ log σ( β log(πθ(T_win|T0)/πref(T_win|T0)) - β log(πθ(T_lose|T0)/πref(T_lose|T0)) ) ]
   ```
   这使学生模型学会偏好视觉可实现细节而非流畅但错误的细节。

- FaithRewriter 在 **所有评估指标**（VQA 得分、文本对齐评分、CLIP/FID、人类偏好排名）上一致优于纯文本重写基线。

- **数据规模**：性能随训练数据增加持续提升，未见饱和。

## 7. 优点

- **创新性**：首次提出“视觉锚点”概念，将提示重写由纯文本任务转化为视觉验证任务，解决了文本数据中难以检测的视觉错误。
- **方法系统性**：构建了“模拟-诊断-修正-验证-蒸馏”的完整闭环，每一步都有明确目标和约束。
- **计算效率**：训练时的高昂开销仅在离线阶段发生，推理时只需一个小型文本 LLM，无额外图像生成或 MLLM 调用。
- **基准质量**：FaithT2I 基准覆盖六种失败类型，使用 VQA 与“None”选项，避免猜测，且经人工验证。
- **实验全面性**：多骨干、多评估器、多生成器、消融、用户研究、规模缩放，验证充分且公平。

## 8. 不足与局限

- **离线计算开销**：数据构建需要大量 T2I 渲染和 MLLM 调用（每候选约 2×2 次），扩展到百万级数据成本高。虽然仅限训练阶段，但仍可能限制大规模实际部署。
- **对底层 T2I 模型能力的依赖**：FaithRewriter 无法克服生成器自身的瓶颈（如无法渲染稀有物体、精确计数、三维翻转等）。若生成器能力不足，增强的提示也无法奏效。
- **实验覆盖范围有限**：仅测试了英文提示，未涉及中文或其他语言；未考虑视频/3D 等扩展任务。
- **潜在偏差风险**：训练数据和评估器均来自类似 deepseek、GPT 等闭源模型，可能存在系统性偏好或数据泄露（尽管声称解耦）。用户研究规模较小（62 人）。
- **未讨论失败案例**：论文未详细分析 FaithRewriter 本身仍然失败的场景（如非常复杂的空间推理）。
