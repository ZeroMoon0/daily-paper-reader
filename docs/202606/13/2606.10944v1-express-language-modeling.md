## 摘要
我们引入了一个新工具 Express，用于将非因果注意力近似转换为具有匹配近似保证的因果近似。当与最先进的 Thinformer 近似相结合时，Express 改进了已知最佳的因果注意力保证，为长度为 n 的序列提供了 $\log^{3/2}(n)/s$ 的近似误差，仅需 $O(s)$ 内存和 $O(s^2 \log^2(n))$ 的压缩开销。我们还将这些进展与高效的 I/O 感知 Triton 实现相结合，展示了相比 FlashAttention 2 的显著加速，并使用 Express 克服了语言建模流程中的四个资源瓶颈：长上下文预填充、KV 缓存压缩、长形式内存受限解码和长形式计算受限解码。

## Abstract
We introduce a new tool, Express, for converting a non-causal attention approximation into a causal approximation with matching approximation guarantees. When combined with the state-of-the-art Thinformer approximation, Express improves upon the best known causal attention guarantees, delivering $\log^{3/2}(n)/s$ approximation error with only $O(s)$ memory and $O(s^2 \log^2(n))$ compression overhead for a sequence of length $n$. We pair these developments with an efficient I/O-aware Triton implementation, demonstrate substantial speedups over FlashAttention 2, and use Express to overcome four resource bottlenecks in the language modeling pipeline: long-context prefill, KV cache compression, long-form memory-constrained decoding, and long-form compute-constrained decoding.

## 2. 方法论：核心思想、关键技术细节、算法流程

**核心思想**：Express是一个可更新的加权缓存对象，它通过结合子高斯瘦化算法（sub-Gaussian thinning）和分层压缩策略，将离线非因果近似转化为流式因果近似。具体过程分为三个阶段：精确阶段（exact phase）、瘦化阶段（thin phase）和削减阶段（halve phase）。瘦化阶段采用均匀子采样（SUBSAMPLE）和压缩算法COMPRESS 2（对分组后的数据逐层应用HALVE）来降低计算开销；削减阶段使用两次HALVE操作将缓存大小重置为目标值。

**关键技术细节**：
- **子高斯瘦化**：算法输出一个加权核集，保证原始数据上的平均函数值近似误差受控于子高斯常数ν。
- **注意力核**：使用键值核κ_att，其形式为exp(⟨k,k'⟩/√d)(⟨v,v'⟩+v_max²)，能同时保证对查询的近似质量。
- **Thinformer Express**：实例化为KH-Express（基于核削减KH算法），核心流程：初始化加权缓存C，对每个新token (q_n, k_n, v_n)，先通过WTD_ATTENTION仅对缓存中的键值对和当前token计算注意力，再更新缓存。查询时间复杂度O(n_out n d)，压缩时间复杂度O(d n_out² log(n_out) log(n/n_out))。

**算法流程（文字说明）**：
1. 初始化缓存为空，目标大小n_out，膨胀因子m，HALVE算法。
2. 精确阶段：前n_out个点直接存入缓存。
3. 瘦化阶段：后续点以批大小2m n_out分组，每批先均匀子采样到2^{m∧m} n_out，再通过COMPRESS 2逐步瘦化到n_out，并入缓存。

## 6. 主要结论与发现

1. **理论优势**：Thinformer Express在因果注意力近似上提供了目前最好的理论保证，误差衰减速率O(n^{-a})，优于HyperAttention的O(n^{-a/6})和BalanceKV的O(1/n_out)，且内存复杂度与n无关。
2. **实际加速**：在长上下文预填充中，512K token时较FlashAttention 2加速82倍（h=9），且在所有层替换后困惑度仅上升1.06倍，同时优于HyperAttention。
3. **KV缓存压缩加速**：结合SnapKV、StreamingLLM、PyramidKV，在几乎不损失准确率的前提下显著缩短运行时间。
4. **长形式解码**：在MATH-500上，Thinformer Express在缓存大小-准确率和时间-准确率双曲线中全面主导五种替代方法，只需61%缓存或56%时间即可达到与精确注意力相同的准确率。
5. **压缩开销极低**：维护缓存的额外时间（~470 ms）远低于查询时间（~35000 ms），证明其实用性。

## 7. 优点

- **强理论保证**：提供了严格可证明的因果注意力近似误差界，且优于现有工作。
- **通用性**：Express可适配任意瘦化算法（即使无理论保证），不限于KH。
- **高效的I/O感知实现**：Triton内核通过分块、并行化、双解引用避免数据拷贝，实现大幅加速。
- **覆盖多场景**：同时解决预填充、KV缓存压缩、内存受限解码和计算受限解码四个瓶颈，实用价值高。
- **低资源需求**：常数量级内存（6n_out），有效支持资源受限设备。

## 8. 不足与局限

- **实验覆盖有限**：仅测试了英语和少量中文（ChatGLM2）语言建模以及数学推理，未涵盖其他自然语言或跨模态任务。
- **实现优化不足**：Triton实现尚未利用warp specialization、Tensor Memory Accelerator、持久化内核或FP8支持（如FlashAttention 2/3已启用），仍有进一步提升空间。
- **仅评估一种基础算法**：虽然Express兼容任何瘦化算法，但实验仅使用了KH-Express，未探索其他基算法的表现。
- **潜在偏差风险**：仅报告了3个LongBench-E任务和MATH-500，任务多样性有限；且模型选择集中在ChatGLM2和Llama 3.1，可能不具完全代表性。
- **环境影响**：虽提及可能降低能源消耗，但也承认效率提升可能导致总使用量增加（杰文斯悖论），未做实际能耗测量。
