## 摘要
推测解码通过让轻量级草稿模型生成候选词，由大型验证器并行验证，从而降低大语言模型的高推理成本。现有的草稿-验证方法采用二元决策：要么接受，要么完全重新计算。然而，我们发现许多被拒绝的token可以通过从完整验证器通过模型内路由导出的一个精简子模型来正确验证，而不是使用完整验证器。这启发我们使用精简验证器来处理需要中等验证资源的token，从而减少对昂贵大型模型的调用。我们提出VIA-SD，一种基于路由的精简验证器的多层级框架。草稿token按层级处理：高置信度情况直接接受，中等置信度情况由精简验证器重新生成，不确定情况由完整模型验证。在四个代表性任务和多个模型家族上，VIA-SD将拒绝率降低了0.10-0.22，比强基线SD方法实现了10-20%的加速，同时比非草稿解码实现了2.5-3倍的加速。此外，VIA-SD与现有SD框架兼容，无需修改其训练过程。我们的结果表明，多层级SD是一种可扩展且高效的大语言模型推理通用范式。项目页面：https://zju-xyc.github.io/VIA-SD-Project-Page/

## Abstract
Speculative decoding (SD) addresses the high inference costs of LLMs by having lightweight drafters generate candidates for large verifiers to validate in parallel. Existing draft-verify methods use binary decisions: accept or fully recompute. Yet we find that many rejected tokens can be verified correctly by a slim submodel derived from the full verifier via intra-model routing, instead of the full verifier. This motivates our slim-verifier to handle tokens requiring moderate verification resources, reducing expensive large-model calls. We propose Verification via Intra-Model Routing for Speculative Decoding (VIA-SD), a multi-tier framework using a routed slim-verifier. Draft tokens are processed hierarchically: direct acceptance for high-confidence cases, slim-verifier regeneration for medium-confidence cases, and full-model verification for uncertain cases. Across four representative tasks and multiple model families, VIA-SD reduces rejection rates by 0.10-0.22 and delivers 10-20% speedups over strong SD baselines, while achieving 2.5-3x acceleration over non-drafting decoding. Moreover, VIA-SD is compatible with existing SD frameworks without modifying their training procedures. Our results suggest multi-tier SD as a general paradigm for scalable and efficient LLM inference. Project page: https://zju-xyc.github.io/VIA-SD-Project-Page/

- **整体含义**：本文提出将推测解码从传统的“两层级”（草稿-验证）范式扩展为“多层级”验证框架（VIA-SD），引入模型内路由（Intra-Model Routing）构建的中间验证器，实现对不同置信度 token 的差异化处理（高置信直接接受、中等置信由精简验证器生成、低置信才调用完整模型），从而在保持或略微提升生成质量的同时显著降低对大模型的调用，实现更高效的 LLM 推理。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 2.1 核心思想
- 基于 **KL 散度**的信息几何视角，将直接的两点分布距离（p → q）分解为多步路径（p → u → q），证明中间分布可以降低总 KL 散度，从而在理论上支持引入中间验证器。
- 利用 **模型内路由**（Intra-Model Routing）从完整验证器 q 中动态选择一部分层（通过路由掩码 z）形成精简子模型 q'，作为中间验证器的实现。这种方法比独立的同规模中间模型更有利于分布一致性，且无需额外加载模型。

### 2.2 关键技术细节
1. **分布度量**：用 KL 散度定性分析，并定义可计算的 KL 风格单步代价函数（式 11），包含接受阈值项和残差替换项，用于评估不同路径的成本。
2. **动态模型内路由（DIMR）**：通过随机搜索与贝叶斯优化相结合的方式，在给定的层跳过比率下，离线搜索最优路由掩码 z*，使 KL 代价最小化。搜索过程单次执行，之后固定用于推理。
3. **三层次验证流程**（算法 1 和算法 2）：
   - **草稿阶段**：轻量级草稿模型 p 生成一个 γ 长度的 token 块。
   - **精简验证器验证**：对每个草稿 token，先用精简验证器 q' 进行验证，设定两个阈值 α1 和 α2：
     - 若 q' 对 token 的置信度 ≥ (1-α1)·max，则直接接受。
     - 若置信度在 (1-α2)·max 与 (1-α1)·max 之间，则由 q' 重新生成该 token。
     - 若置信度 < (1-α2)·max，则升级到完整验证器 q 处理。
   - **完整模型验证**：仅对最不确定的 token 调用完整模型 q，进行接受或重新生成。
4. **混合目标分布**：将三个阶段的输出通过权重 δ1 和 δ2 组合为混合分布 π，保证一致性。

## 6. 论文的主要结论与发现

1. **多层级验证优于二元验证**：VIA-SD 在几乎所有实验中降低了拒绝率（0.10～0.22 绝对下降），实现了比强 SD 基线 10-20% 的加速，比非草稿解码 2.5-3 倍加速。

3. **兼容现有框架**：VIA-SD 可以直接集成到 EAGLE、PEARL 等现有 SD 系统中，无需修改训练流程，进一步提升了它们的性能（拒绝率降低 20-31%，加速提升 8-26%）。
4. **质量不降反升**：在部分任务（如问答、代码）中，由于损失性验证引入了有益的正则化效果，VIA-SD 有时甚至略微超过了直接使用完整模型贪婪解码的质量。
5. **超参数具有鲁棒性**：默认设置（α1=0.5, α2=0.3, 跳过比率 45%, γ=5）在多个场景下接近最优，且性能随超参数变化平滑退化。

## 7. 优点

- **理论创新**：将推测解码从信息几何（KL 散度）角度重新审视，为引入中间验证器提供了形式化的动机。
- **实用性**：提出的 DIMR 方法仅需一次离线搜索（不到 1 GPU 小时），即可获得固定的精简验证器，推理时无额外开销；且无需训练模型、无需修改模型参数。
- **模块化设计**：精简验证器仅改变验证流程，不干涉草稿模型或完整模型的结构，因此可以即插即用到任意草稿-验证对中。
