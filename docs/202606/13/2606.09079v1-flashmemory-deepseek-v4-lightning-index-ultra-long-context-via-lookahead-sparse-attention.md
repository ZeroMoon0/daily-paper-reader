## 摘要
传统LLM在解码时保持完整的KV缓存，导致超长上下文服务出现严重的GPU内存瓶颈。在本报告中，我们提出了前瞻稀疏注意力（LSA），这是一种基于DeepSeek-V4架构构建的神经内存索引器驱动的全新推理范式。LSA不是被动地关注所有历史令牌，而是主动预测未来的上下文需求，并仅在GPU内存中保留查询关键的KV块。关键的是，我们通过无主干解耦训练策略实例化了这一架构。通过将索引器制定为标准双编码器架构，我们使用标准检索训练框架独立训练它，而无需将庞大的主干模型加载到GPU内存中。我们证明，这种“少即是多”的范式显著最大限度地提高了服务效率，同时在依赖长期全局记忆的任务中充当有效的注意力去噪器。在主要的长上下文评估套件（例如LongBench-v2、LongMemEval和RULER）中，FM-DS-V4将平均物理KV缓存占用压缩到全上下文基线的仅13.5%，同时一致地保持或略微提升下游准确性（平均绝对边际+0.6%）。关键的是，在极端500K规模下，FlashMemory将物理KV缓存开销降低了90%以上，而不会破坏主干核心推理能力的稳定性。

## Abstract
Conventional LLMs keep the full KV cache loaded during decoding, causing a severe GPU memory bottleneck for ultra-long context serving. In this report, we propose Lookahead Sparse Attention (LSA), a novel inference paradigm powered by a Neural Memory Indexer built upon the DeepSeek-V4 architecture. Rather than passively attending to all historical tokens, LSA proactively predicts future context demands and preserves only the query-critical KV chunks in the GPU memory. Crucially, we instantiate this architecture via a backbone-free decoupled training strategy. By formulating the indexer as a standard dual-encoder architecture, we train it independently using standard retrieval training frameworks without ever loading the massive backbone model into GPU memory.   We demonstrate that this "less is more" paradigm significantly maximizes serving efficiency while acting as an effective attention denoiser in tasks that rely on long-term global memory. Across primary long-context evaluation suites (e.g., LongBench-v2, LongMemEval, and RULER), FM-DS-V4 compresses the average physical KV cache footprint down to merely 13.5% of the full-context baseline, while consistently preserving or slightly elevating downstream accuracy (+0.6% absolute margin on average). Crucially, at extreme 500K scales, FlashMemory suppresses the physical KV cache overhead by over 90% without destabilizing the backbone's core reasoning capacities.

## 2. 方法论

### 核心思想
- **Lookahead Sparse Attention (LSA)**：在DeepSeek-V4架构基础上，增加一个轻量级**神经内存索引器（Neural Memory Indexer）**，该索引器每固定间隔τ步（τ=64）触发一次，根据当前查询的隐藏状态，预测未来窗口（τ步内）需要哪些历史压缩KV块，并仅将这些块从CPU冷池提取到GPU内存，其余历史块存储在CPU上。

### 关键技术细节
1. **索引器架构**：采用与DeepSeek-V4原生Lightning Indexer相同的双编码器结构，但将最后一层激活函数从ReLU替换为**Sigmoid**（输出范围0~1），并用**阈值（0.5）** 代替固定的Top-k选择，实现动态召回数量。
   - 查询编码：通过低秩投影（下投影W_DQ + 上投影W_IU_Q）得到多头索引查询；并通过路由权重头wlt动态缩放各头重要性。
   - 匹配分数：头融合后的门控匹配分数为 \( I_{t,s} = \sigma\left( \sum_h w_{t,h} \cdot \text{ReLU}(q_{t,h} \cdot (K^{Comp}_s)^T) \right) \)。
   - 选中的块：\( C^{MemComp}_t = \{ C^{Comp}_s \mid I_{t,s} \ge 0.5 \} \)。
2. **层级配置**：最终选择**第10、12、20层**的三层索引器，采用**OR模式**（任一层的分数≥0.5即召回），达到性能与效率的最佳平衡。
3. **训练数据构建（Lookahead Dataset）**：
   - 对每个解码步骤i，提取所有CSA层的原始索引器logit，通过**三步去噪管道**生成黄金标签：
     - Softmax归一化 → Top-p核采样（p=0.6） → **跨层多数投票**（至少3层同时选中才视为正标签）。
   - 最终正标签为未来τ步内黄金标签的并集。
4. **解耦训练策略**：
   - 索引器作为独立双编码器，仅训练三个投影矩阵（W_DQ, W_IU_Q, W_w），**完全冻结**骨干模型和预计算的压缩索引键。
   - 训练损失：**二元交叉熵（BCE）** 基础上引入**Focal Loss**（γ=2）以抑制易分负样本；负采样比例3:1。
   - 训练过程中无需加载千亿参数骨干模型，计算量极低。

## 3. 实验设计

### 数据集
- **LongBench-v2**：S（46K）、M（179K）、L（493K）三个规模。
- **LongMemEval**：S（125K）、M（500K）。
- **RULER**：64K、128K、256K、512K四个长度。
- 另外还构建了**上下文无关（No-Context）** 对抗任务，以及**MRCR**（多范围上下文检索）基准。

### Benchmark与对比方法
- **DS-V4-Flash**：标准DeepSeek-V4-Flash模型（全KV缓存）。
- **Recency Only**：仅保留最近8K tokens的滑动窗口。
- **Random 10%**：随机保留10%历史块。
- **FM-DS-V4（Ours）**：本文提出的LSA方法。

### 实验结果摘要
- **平均GPU内存**：FM-DS-V4仅占DS-V4-Flash的**13.5%**（平均从3.90GB降至0.42GB），极端500K下内存降低**90%**。
- **平均准确率**：FM-DS-V4为77.5%，比DS-V4-Flash的76.9%**提升0.6个百分点**。
- **Recency Only和Random 10%** 在所有全局依赖任务上崩溃（如LongMemEval-S仅19.2%），证实索引器学会了有效的时间路由。

## 6. 主要结论与发现

- **“少即是多”**：LSA在降低86.5%内存占用的同时，平均准确率反而提升0.6%，表明**注意力去噪**效果——过滤掉历史无关块，减少注意力点积中的噪声。
- **在极端500K长度下**，内存节省超过90%，性能保持甚至优于全基线。
- **索引器需在训练长度2倍内**才能保持有效性；超出则退化为随机。
- **MRCR任务**是当前方法的死角（准确率从76%降至48%），说明需要更深的交互机制（如ColBERT式晚交互）才能应对密集全局检索。

## 7. 优点

- **创新性**：将**检索式索引器**引入LLM推理，实现预测性稀疏注意力，打破了传统全量KV缓存的内存瓶颈。
- **极低训练成本**：解耦训练策略使得索引器可以在1个GPU小时内完成优化，无需任何骨干模型参与，大幅降低资源需求。
- **高效迭代**：可在短时间内完成数百次实验以探索最优架构。
- **诚实透明**：明确报告了失败案例（MRCR、上下文无关内存非恒定、长度泛化限制），并分析了根本原因。
- **实用性强**：在主流长上下文基准上取得了显著的内存压缩效果，同时不牺牲性能。

## 8. 不足与局限

- **未系统消融关键超参数**：τ、阈值、层数等均未进行严格对比实验，最优性未完全验证。

  - 冻结的key表示限制了表征能力。
