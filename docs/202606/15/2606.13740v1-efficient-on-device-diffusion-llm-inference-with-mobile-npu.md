## 摘要
扩散大语言模型（dLLM）通过并行去噪多个token来加速生成，使其对延迟敏感的移动推理具有吸引力。然而，重复去噪在智能手机上引入了大量计算。移动神经处理单元（NPU）提供高吞吐量的密集矩阵计算，但高效利用它们仍面临挑战：token承诺使每块有效工作负载缩小，token修订使KV缓存复用复杂化，且有限的NPU可见地址空间导致昂贵的重映射和数据传输开销。本文提出llada.cpp，这是首个在智能手机上加速dLLM的NPU感知推理框架。llada.cpp通过三种技术将分块dLLM推理与移动NPU的执行特性对齐：(1) 多块投机解码，使用投机的未来块token填充当前块解码后期缩小的工作负载；(2) 双路径渐进修订，使已承诺的token在稳定前保持可修订，并通过CPU侧路径刷新不稳定token，而不会阻塞密集的NPU执行；(3) 交换优化内存运行时，压缩NPU可见地址布局，并将数据准备与NPU计算重叠，以减少重映射和传输开销。我们将llada.cpp实现为端到端框架，并在多种硬件平台和dLLM工作负载上对其进行评估。与使用前缀KV缓存复用的CPU基线相比，llada.cpp将LLaDA-8B的生成延迟降低了17倍至42倍，同时保持了生成质量。

## Abstract
Diffusion large language models (dLLMs) accelerate generation by denoising multiple tokens in parallel, making them attractive for latency-sensitive mobile inference. However, repeated denoising introduces substantial computation on smartphones. Mobile neural processing units (NPUs) offer high-throughput dense matrix computation, but efficiently exploiting them remains challenging: token commitment shrinks per-block effective workloads, token revision complicates KV cache reuse, and limited NPU-visible address space incurs costly remapping and data transfer overheads.   In this paper, we propose llada.cpp, the first NPU-aware inference framework for accelerating dLLMs on smartphones. llada.cpp aligns block-wise dLLM inference with the execution characteristics of mobile NPUs through three techniques. (1) Multi-Block Speculative Decoding fills the shrinking workload in late-stage current-block decoding with speculative future-block tokens. (2) Dual-Path Progressive Revision keeps committed tokens revisable until stable and refreshes unstable tokens through a CPU-side path without stalling dense NPU execution. (3) Swap-Optimized Memory Runtime compacts NPU-visible address layouts and overlaps data staging with NPU computation to reduce remapping and transfer overheads. We implement llada.cpp as an end-to-end framework and evaluate it across diverse hardware platforms and dLLM workloads. llada.cpp reduces LLaDA-8B generation latency by 17x-42x over the CPU baseline with prefix KV cache reuse, while preserving generation quality.

## 1. 核心问题与整体含义（研究动机和背景）

扩散大语言模型（dLLM）通过并行去噪多个token来加速生成，有望降低移动设备上的延迟瓶颈。然而，重复去噪在智能手机上引入大量计算；移动神经处理单元（NPU）虽能高效执行密集矩阵运算，但存在三个关键问题：
- **token承诺**：去噪后期剩余掩码token减少，导致NPU有效工作负载萎缩；
- **token修订**：已承诺token仍需修订，使KV缓存复用复杂化；
- **内存访问**：NPU可见地址空间有限，频繁数据重映射和传输带来高昂开销。

因此，**如何将动态的块式dLLM推理与静态的移动NPU执行模型对齐**，是该论文的核心研究问题。

## 2. 方法论

论文提出 **llada.cpp**，首个NPU感知的dLLM推理框架，包含三项关键技术：

### 2.1 多块投机解码（Multi-Block Speculative Decoding）
- **动机**：当前块解码后期剩余token少，NPU利用率低。
- **操作**：将当前块“解码窗口”向右滑动，并主动纳入未来块的token作为“草稿”参与NPU前向传播，增加有效并行工作量。
- **策略**：
  - **连续滑动**：根据token承诺前沿自然右移窗口。
  - **自适应前视**：根据NPU延迟分桶特性，动态决定扩展未来token数量，避免触发下一个延迟桶。

### 2.2 双路径渐进修订（Dual-Path Progressive Revision）
- **三态机制**：
  - **不可见**：掩码或低置信度token。
  - **可见**：有具体预测值但可能不稳定，可用于上下文但不永久合并。
  - **稳定**：通过更高阈值判定，永久合并入前缀KV缓存。
- **异步CPU刷新路径**：
  - 使用专用CPU核直接解码HMX布局的权重，在NPU执行密集去噪的同时，异步更新不稳定token的KV状态。
  - 采用**延迟合并**策略，仅在步间边界合并刷新结果，避免细粒度同步。
- **选择性logits跳过**：对已稳定token跳过输出投影和采样，减少CPU侧计算。

### 2.3 交换优化内存运行时（Swap-Optimized Memory Runtime）
- **图引导缓冲区映射**：
  - 根据计算图和算子分析，将张量分类为长寿命（权重、KV缓存）和短寿命（激活、临时数据）。
  - 使用profile-guided placement决定哪些数据常驻NPU虚拟地址空间，哪些通过暂存区映射。
- **流水线式数据传输**：
  - 双缓冲暂存区：在NPU运算同时，预准备下一个算子的缓冲区描述符并验证映射。
  - 重叠数据准备时间与NPU计算时间，隐藏不可避免的交换开销。

> **总体流程**：块式解码→多块投机窗口确定→NPU前向（密集）+ CPU异步修订（稀疏）→步间刷新→输出稳定token。

## 6. 主要结论与发现

1. **llada.cpp大幅降低延迟**：在SM8750上，LLaDA-8B生成128 token较CPU基线加速17×–42×（使用前缀KV缓存时）。
2. **超越自回归模型**：在SM8750/SM8850上，llada.cpp运行LLaDA-8B速度**快于**等规模Llama-3-8B的NPU加速版本（部分设置高达3.9×）。
3. **质量保持**：llada.cpp在四个数据集上的准确率与CPU基线相当甚至略优，表明所提优化未导致质量退化。
4. **能量效率改进**：llada.cpp平均功率和每请求能量均低于CPU基线，而NPU加速的Llama因并行度不足反而功耗更高。
5. **组件贡献**：最大加速来自分段token稳定（STS，3.65×）和多块投机解码（MBSD，1.20×）；交换优化内存运行时（SOMR）提供额外1.22×，并支持更长的输出场景。

## 7. 优点

- **系统性**：首次完整解决dLLM在移动NPU上从token管理到内存访问的全栈问题，而非单一优化点。

- **实用性**：依托llama.cpp实现开源生态，可直接在现有手机上运行；支持多代高通SoC。
- **实验严谨**：控制变量充分，消融实验清晰显示各组件贡献，且考虑了能量维度。
