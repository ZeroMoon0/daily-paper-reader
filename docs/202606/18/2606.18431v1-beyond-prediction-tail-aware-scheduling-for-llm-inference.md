## 摘要
LLM服务表现出极端的长度变异，使得基于大小的调度在实践中变得困难。最近的LLM调度器使用预测的解码长度或排名来近似SJF/SRPT，并主要报告以均值为主的指标如TTFT和TBT。我们表明，这些预测驱动的策略在分布偏移、突发到达和GPU内存压力下可能变得脆弱，同时，即使拥有完美的解码长度知识，对主导用户体验的尾延迟（P90-P99）控制也有限。我们引入了一种分布感知、无需预测的调度框架，用由轻量级统计信号驱动的软优先级提升取代了显式长度预测。我们的设计协同优化了调度和缓存感知的抢占，以考虑跨工作负载混合的内存耦合解码动态。在生产环境和开源轨迹上的评估表明，相对于具有完美长度知识的SRPT，我们的方法将P99 TTLT降低了高达35-50%，并且跨工作负载（包括推理密集型任务和聊天密集型任务）将TTFT降低了34-47%。这些结果证明了在线LLM服务中优化尾延迟的一种稳健替代方案。

## Abstract
LLM serving exhibits extreme length variability, making size-based scheduling difficult in practice. Recent LLM schedulers approximate SJF/SRPT using predicted decode lengths or ranks and primarily report mean-centric metrics such as TTFT and TBT. We show that these prediction-driven policies can be fragile under distribution shifts, bursty arrivals, and GPU memory pressure, while offering limited control over the tail latency (P90-P99) that dominates user experience, even with perfect decode-length knowledge. We introduce a distribution-aware, prediction-free scheduling framework that replaces explicit length prediction with soft priority boosting driven by lightweight statistical signals. Our design co-optimizes scheduling and cache-aware preemption to account for memory-coupled decode dynamics across workload mixes. Evaluated on production and open-source traces, our method reduces P99 TTLT by up to 35-50% relative to SRPT with perfect length knowledge and reduces TTFT by 34-47% across workloads, including reasoning-heavy and chat-heavy tasks. These results demonstrate a robust alternative for optimizing tail latency in online LLM serving.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 推理服务面临极端请求长度变异性（特别是解码长度），导致基于预测的调度策略（如 SJF/SRPT）在分布偏移、突发到达和 GPU 内存压力下表现脆弱。同时，即使拥有完美的解码长度预测，这类调度器对用户感知至关重要的尾延迟（P90–P99）控制仍然有限。
- **研究背景**：现有 LLM 调度器多通过预测输出长度或排名来近似 SJF/SRPT，并主要报告均值指标（如平均 TTFT、TBT）。然而均值优化不保证尾延迟优化，且 LLM 推断是**状态有记忆、内存耦合**的过程（KV cache 积累），使得预取和抢占代价高昂。论文因此提出了**预测无关、分布感知的尾延迟优化调度框架**，旨在不依赖长度预测的前提下提升 P95/P99 TTLT 和 TTFT。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：采用软优先级提升（Boost）替代硬性基于大小的排序，用轻量级统计信号（如已解码 token 数、历史请求延迟分布）计算每个请求的“提升值（boost）”，然后按“到达时间减去提升值”排序。这种方法可平滑地在 FCFS 和短任务优先之间插值，避免头部阻塞和饥饿。
- **关键技术细节**：
  1. **γ-Boost 优先函数**：对请求 `i` 在时刻 `t` 定义优先级 `ϕ_i(t) = a_i - b_γ(w_i(t))`，其中 `b_γ(w) = (1/γ) log(1/(1 - e^{-γ w}))`。`γ` 控制策略偏向短任务（小γ）或长任务（大γ）。该策略在理论上可实现渐近最优尾延迟。
  2. **统一优先空间（UniBoost）**：将预填充和解码阶段统一到一个优先级空间中，请求的“有效已服务量”定义为 `̃w_i(t) = max(wi(t), s_i^{pre})`，避免分阶段头阻塞。
  3. **MemGuard 缓存感知抢占**：引入几何量化粒度 `k`，只有当请求跨越量化阈值时才重新计算优先级并允许抢占。这限制了一次请求的生命周期内最多发生 `O(log S_i/k)` 次优先级变更，减少 KV 缓存抖动。
  4. **自适应 γ 估计（γ-Ada）**：在线监测 P95 和 P99 延迟，拟合尾部分布的指数衰减率，平滑更新 `γ`，以适应负载分布变化。
- **算法流程**（文字说明）：
  - 维护两个堆（预填充队列和解码队列），每个请求的优先级由 `a_i - boost(当前有效已服务量)` 计算。
  - 每步从两个堆中选出优先级最高的请求（跨阶段统一排序），检查是否满足 KV 容量；若不满足则调用 MemGuard 换出最低优先级请求。
  - 执行一个微批次（先解码再预填充），更新统计信息并更新 γ。

## 7. 优点

- **理论驱动**：从队列论中尾延迟最优调度（γ-Boost）出发，给出了在 M/G/1 下饥饿界限的证明，并扩展到实际 LLM 场景中的内存耦合约束。
- **无需预测**：避免了对模型输出长度的显式预测或学习排序，从而在本质上更鲁棒于预测误差和分布偏移。
- **协同设计**：同时优化调度决策和 KV 缓存抢占，避免了传统 SRPT 可能带来的频繁上下文切换和缓存抖动，将预取开销控制在 `O(log S)` 级别。
- **自适应与通用性**：通过在线估计 γ，能够自动适应负载和分布变化；此外，可自然扩展支持前缀缓存、连续批处理和全局路由（如 JSQ），实验验证了良好的组合效果。
- **实验全面**：覆盖了多模型、多负载、多指标、消融和敏感性分析，并验证了可扩展性。

## 8. 不足与局限

- **理论分析的简化**：算法的强尾最优性证明基于 M/G/1 队列假设，但 LLM 实际服务更接近 M/G/k（连续批处理）。当前 M/G/k 下的 Boost 理论尚不完整，论文对 k>1 的扩展主要依赖实验验证，缺乏严格证明。
- **实验规模有限**：最大算力测试为 8 块 A100（单节点），集群扩展实验采用轨迹回放（trace-replay）方法，未在真实多节点生产集群上实际部署验证。GPU 型号单一（仅 A100），未测试不同架构（如 H100、AMD MI300）。
- **内存约束处理的局限**：MemGuard 以几何块大小限制抢占次数，但预填充和解码的实际 KV 缓存需求差异大，可能不是最优策略；并且当工作负载中长请求比例极高时，即使是 O(log S) 的抢占也可能导致显著开销。
- **依赖于手动设定分块大小 `k` 和迟滞 `δ`**：虽然论文提供了敏感性分析，但未给出自适应调整这些参数的方法，跨集群部署时可能需要重新调参。
- **未考虑系统性扰动**：实验中假设 GPU 内存容量固定、无其他租户干扰，但生产环境中可能存在共享 GPU 上其他任务的内存或计算竞争，这可能会影响调度效果。
- **公平性度量局限**：仅测试了一种混合比例下的饥饿情况，未覆盖更极端的优先级反转场景（如极高的长请求比例或自旋循环），尽管理论给出了无饥饿的上界，但实践中可能需要更多验证。
