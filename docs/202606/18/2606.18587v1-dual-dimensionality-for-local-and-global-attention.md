## 摘要
仅解码器Transformer会计算前面token的KV缓存的注意力。键（和值）通常用相同的维度表示，无论其与预测目标的距离如何。然而，在自然语言中，下一个单词受紧邻的前面token影响最大。我们假设局部和远处的token对表示能力有不对称的需求：局部token对于预测即时输出更为关键，因此需要更丰富的表示，而远处token主要作为长程记忆，低维表示可能就足够了。我们将这一思想形式化为距离自适应表示（DAR），在受控设置中实现：在局部上下文窗口内保留全维表示，而对该窗口之外的token分配降维表示（例如原维度的1/4）。在多个预训练规模（70M到410M参数）以及1B规模模型上的持续监督微调中，该方法与全维基线的性能紧密匹配。相比之下，在所有token位置上均匀降低维度会导致性能下降。这些结果挑战了键和值维度应跨token位置均匀的常见假设。我们的发现为设计注意力架构提供了新方向，该架构可以在序列中自适应地分配表示能力，从而在推理期间进一步减少KV缓存。

## Abstract
Decoder-only Transformers compute attention over the KV cache of preceding tokens. Keys (and Values) are typically represented with the same dimensionality, regardless of its distance from the prediction target. In natural language, however, the next word is most strongly influenced by the immediately preceding tokens. We hypothesize that local and distant tokens impose asymmetric demands on representational capacity: local tokens are more critical for predicting immediate outputs and thus require richer representations, whereas distant tokens primarily serve as long-range memory, for which lower-dimensional representations may suffice. We formalize this idea as Distance-Adaptive Representation (DAR), implemented in a controlled setting that preserves full-dimensional representations within a local context window while assigning reduced-dimensional representations (e.g. 1/4 of the original dimensionality) to tokens beyond that window. Across multiple pretraining scales (70M to 410M parameters), as well as continued supervised fine-tuning on a 1B-scale model, this approach closely matches the performance of full-dimensional baselines. In contrast, uniformly reducing dimensionality across all token positions leads to worse performance. These results challenge the common assumption that key and value dimensionality should be uniform across token positions. Our findings suggest a new direction for designing attention architectures that adaptively allocate representational capacity across sequences, enabling further reductions in KV cache during inference.

## 论文详细总结（自动生成）

### 1. 论文的核心问题与整体含义（研究动机和背景）

当前 decoder-only Transformer 在计算注意力时，对**所有历史 token 的 Key 和 Value 都采用相同的维度**（即全维表示），无论该 token 距离当前预测位置有多远。然而在自然语言中，**紧邻的局部 token 对下一个词的预测影响最大**（如避免立即重复、遵守局部语法等），而**远距离 token 主要提供长程语义记忆**，对瞬间细节的敏感度较低。  
论文基于这一观察提出假设：**局部 token 需要高维表示来承载细粒度信息，远距离 token 则可以容忍更低维的表示**。验证这一假设有助于设计出更高效的注意力架构，在推理时减少 KV 缓存大小，同时保持性能。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：**距离自适应表示（Distance-Adaptive Representation, DAR）**——根据 token 与预测目标的距离动态分配表示维度。
- **具体实现（两区域划分方案）**：
  1. 保留一个**局部窗口**（大小为 \(w\)），窗口内的 token 使用原始全维隐藏状态 \(h_j \in \mathbb{R}^d\)。
  2. 窗口外的 token 先通过一个轻量线性下投影 \(W_{\text{down}} \in \mathbb{R}^{d \times d_{\text{down}}}\) 得到低维表示 \(h_j^D = h_j W_{\text{down}}\)，再通过上投影 \(W_{\text{up}} \in \mathbb{R}^{d_{\text{down}} \times d}\) 恢复成与 KV 投影兼容的维度 \(h_j'\)。
  3. 注意力计算时，根据距离选择键值：
     - 若 \(i - j < w\)（局部），使用 \(k_j = \text{RoPE}(h_j W_K), v_j = h_j W_V\)。
     - 否则（全局），使用 \(k_j = \text{RoPE}(h_j' W_K), v_j = h_j' W_V\)。
  4. 训练采用标准下一 token 预测损失，无额外辅助损失；所有参数（包括 \(W_{\text{down}}, W_{\text{up}}\)）通过反向传播更新。
  5. 推理时可进一步优化：仅缓存 \(h_j^D\)（低维），在需要时按需计算 KV，将内存复杂度从 \(O(Td)\) 降至 \(O(T d_{\text{down}} + w d)\)。

### 3. 实验设计：数据集、benchmark、对比方法

- **预训练实验**：
  - **数据集**：10B token 的 Pile 子集；评估用 FineWeb-Edu、C4、WikiText-103。
  - **模型**：Pythia-70M、Pythia-160M、Pythia-410M（序列长度 2048）。
  - **对比方法**：
    - Vanilla：全维度标准注意力（d=512/768/1024）。
    - 均匀降维（Uniform reduction）：对所有 token 使用同一低维 \(d_{\text{down}}\)（无局部/全局区分）。
    - DAR：不同的 \(d_{\text{down}}\)（如 256、128、64）和不同窗口大小 \(w\)。
- **监督微调实验**：
  - **数据集**：OLMo-2-1B-SFT（基于 Tülu 3 的数据），两阶段训练（先冻住主干只训练瓶颈参数，再全参数训练）。
  - **评估**：6 个下游 benchmark：MMLU（多任务理解）、HellaSwag（常识推理）、CommonsenseQA（常识问答）、GSM8K（数学推理）、MBPP（代码生成）、Multi-News（长文摘要）。
  - **基线**：使用同一 DAR 架构但 \(d_{\text{down}} = d = 2048\)（无实际降维）作为 no-bottleneck baseline。

### 6. 论文的主要结论与发现

- DAR 在 \(d_{\text{down}} = d/2\) 或 \(d/4\) 时**性能与全维度基线相当甚至更好**（如 70M 下相对困惑度 99.61%，410M 下 97.98%）。
- 均匀降维在所有设置下均明显劣于 DAR，说明**局部 token 确实需要更高维表示**。
- 窗口大小在较宽范围（\(w \ge 4\)）内对性能影响很小，表明**仅需非常小的局部窗口提供全维能力**。
- 下游任务在 \(d_{\text{down}} \ge d/8\) 时性能保持，低于该阈值时显著下降。
- 这些结果**挑战了 KV 维度必须统一的假设**，为设计自适应分配表示能力的注意力架构提供了证据。

### 7. 优点

- **方法简单有效**：仅需在下投影+上投影的线性变换，无需改变注意力计算核心，易于集成。
- **实验设计清晰**：同时对比“DAR vs 均匀降维”和“DAR vs 全维度”，严格隔离距离因素；消融实验充分（窗口大小、bottleneck 维度、模型规模）。
- **对推理效率有启发**：指出可通过吸收投影矩阵来按需计算 KV，将远距离 token 缓存从 \(O(d)\) 降到 \(O(d_{\text{down}})\)，潜力巨大。

### 8. 不足与局限

- **当前实现存储双重 KV**：训练和推理实验中仍保留了全维和低维两组 KV，导致缓存翻倍，与降低缓存的初衷相悖。作者在 §5 中说明了高效推理方案，但未实际验证。
- **模型规模有限**：预训练最大 410M 参数，微调最大 1B；更大型模型（如 7B+）上的表现未知。
- **架构覆盖**：仅测试了标准 decoder-only Transformer，未验证 MLA、GQA 等变体下的适配性。
- **距离自适应机制较粗糙**：采用两区域硬划分，未探索随距离逐渐降低维度的连续方案。
- **下游任务评估中未包含长程依赖任务**（如书籍理解、多轮对话），难以全面判断全局路径在长序列中的有效性。
