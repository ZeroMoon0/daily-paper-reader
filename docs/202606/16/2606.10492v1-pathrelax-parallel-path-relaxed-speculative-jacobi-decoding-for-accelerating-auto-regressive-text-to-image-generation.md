## 摘要
自回归文本到图像模型对高分辨率图像生成的需求日益增长，导致令牌序列变长，显著增加了计算成本和推理时间。然而，现有的加速自回归文本到图像模型的最先进方法依赖于链式结构的草稿令牌序列，导致草稿令牌搜索效率低下且接受长度有限。为了解决这个问题，我们提出了并行路径交叉松弛推断式雅可比解码（PathSpec），一种通过多序列草稿树结构提升效率的新框架。我们的并行路径推断式雅可比解码（PathExplore）扩展了令牌搜索空间，在不牺牲图像质量的情况下实现了更高的加速比。此外，我们引入了跨路径松弛验证（PathRelax），利用序列间的语义相似性进一步提高令牌接受率。在Parti-Prompts、MSCOCO2017和T2ICompBench数据集上的评估表明，我们的方法分别实现了4.14倍、3.95倍和4.18倍的加速比。值得注意的是，PathExplore在没有任何松弛采样的情况下，其加速比优于GSD和LANTERN等松弛采样方法。此外，PathRelax的松弛机制可以与其他松弛技术无缝集成，实现进一步加速，为实时文本到图像生成提供了一种高效的解决方案。我们的代码可在https://github.com/Haodong-Lei-Ray/PathSpec获取。

## Abstract
The growing need for high-resolution image generation in autoregressive text-to-image models has resulted in extended token sequences, significantly increasing computational costs and inference times. However, existing state-of-the-art methods for accelerating autoregressive text-to-image models rely on chain-structured draft token sequences, leading to inefficient draft token search and limited acceptance lengths. To address this, we propose parallel-path cross-relaxed speculative Jacobi decoding (\textbf{PathSpec}), a novel framework that enhances efficiency through a multi-sequence draft tree structure. Our parallel-path speculative Jacobi decoding (\textbf{PathExplore}) expands the token search space, achieving a higher speedup ratio without sacrificing image quality. Additionally, we introduce cross-path relaxed verification (\textbf{PathRelax}) that exploits semantic similarities across sequences to further boost token acceptance rates. Evaluated on the Parti-Prompts, MSCOCO2017, and T2ICompBench datasets, our method achieves a speedup ratio of 4.14 $\times$, 3.95$\times$, and 4.18$\times$, respectively. Remarkably, PathExplore, without any relaxed sampling, outperforms relaxed sampling methods in the speedup ratio, such as GSD and LANTERN. Moreover, PathRelax's relaxation mechanism can be seamlessly integrated with other relaxation techniques, enabling further acceleration and providing an efficient solution for real-time text-to-image generation. Our code is available at https://github.com/Haodong-Lei-Ray/PathSpec.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：自回归（AR）文本到图像生成模型（如 Lumina-mGPT、Emu3）需要逐 Token 生成长达数千的离散图像 Token 序列，导致推理延迟极高，难以满足实时生成需求。
- **现有局限**：现有的推测解码加速方法依赖**链式草稿序列**（如 SJD），链中早期 Token 被拒绝会导致后续草稿失效，限制接受长度；松弛策略（如 GSD、LANTERN）仅在单 Token 的代码本空间进行局部放松，忽略跨分支的语义结构。
- **目标**：设计一种能扩大搜索空间、利用分支间语义相似性提升接受率的加速框架，同时保持生成质量无损（或可接受的质量损失）。

## 2. 方法论

### 2.1 核心思想
- 提出 **PathSpec** 框架，包含两个关键组件：
  - **PathExplore**：将单链草稿扩展为**多序列草稿树**，多个候选路径并行生成和验证，避免链式依赖导致的级联拒绝。
  - **PathRelax**：在草稿树中引入**跨路径概率聚合**，利用同一父节点下不同子 Token 的语义相似性，软化验证准则，进一步提升接受率。

### 2.2 关键技术细节
- **草稿树构建**：
  - 不使用外部草稿模型，通过随机初始化根节点和自回归预测生成候选 Token。
  - 每个迭代中，基于已接受前缀并行生成多个子候选，形成树结构。树参数由深度 \(W\) 和每层分支数 \(B\) 控制，总节点限制为 72。
- **验证机制**：
  - 对所有草稿路径同时进行并行验证，采用标准拒绝采样准则（式 (2)）。
  - PathRelax 扩展验证：对同一父节点的所有子 Token，计算**跨路径聚合概率**（式 (7)）：
    \[
    p_{\text{target}}(x_{L,k}) = p_{\theta}(\cdot) + \lambda \cdot \frac{1}{K-1} \sum_{s\neq k} p_{\theta}(\cdot)
    \]
    其中 \(\lambda\) 控制松弛强度。然后使用该聚合概率进行拒绝采样（式 (8)），从而允许语义相似的边缘 Token 更容易被接受。
- **等级选择**：每轮迭代后，选择最长公共前缀最长的已接受序列作为输出，其余候选节点按等级限制更新到下一轮草稿树，不足的节点用随机初始化补充。

- **质量退化**：引入松弛后（PathRelax 或 λ>0），图像质量指标有轻微下降（CLIP、HPSv2 降低 3%–8%），对于高保真需求（如人像）可能不可接受。论文未提供人类评估量化感知损失。

- **应用限制**：对要求严格无偏生成的场景（如医疗图像），PathRelax 可能因松弛引入分布偏移，需要谨慎使用。
