## 摘要
4位量化显著降低了内存占用并加速了大语言模型（LLM）的推理。然而，其有限的位宽表示难以同时忠实捕获密集的常见值（内点）和罕见的大幅值（离群点），导致严重的精度下降。现有的混合精度方法通过将离群点保留在高精度来缓解这一问题，但代价是破坏了低位执行的统一性，引入了精度转换和额外数据移动，从而削弱了实际加速效果。我们提出MosaicQuant，一种基于内点-离群点分解这一新原理的统一4位LLM量化范式。MosaicQuant并非提升离群点精度，而是将整个权重矩阵量化成一个密集的4位基分量，其中内点被忠实捕获，而离群点则不可避免地被量化。随后引入一个稀疏的4位残差分量来补偿这些量化误差，选择性地针对输出失真被证明最集中的误差关键权重块。然而，仅统一表示是不够的，因为将稀疏残差作为单独核执行仍会破坏统一的低位推理流水线。为解决这一问题，我们引入ZipperEngine，它通过重叠流水线将稀疏块计算融合到密集4位GEMM核中，不仅统一了表示，而且将执行统一为单个连贯的低位推理流水线。在LLaMA3和Qwen3上的大量实验表明，MosaicQuant在保持接近FP16精度的同时，相比W16A16基线实现了高达1.24倍的加速。

## Abstract
4-bit quantization significantly reduces the memory footprint and accelerates the inference of large language models (LLMs). However, its limited bit-width representation struggles to faithfully capture both dense common values (\emph{inliers}) and rare large-magnitude values (\emph{outliers}), causing substantial accuracy degradation. Existing mixed-precision methods mitigate this by retaining outliers in high precision, but at the cost of breaking the uniformity of low-bit execution, introducing precision conversion and extra data movement that undermine practical speedup. We propose \textbf{MosaicQuant}, a unified 4-bit LLM quantization paradigm built on a novel principle of \emph{inlier--outlier disaggregation}. Rather than elevating outlier precision, MosaicQuant quantizes the full weight matrix into a dense 4-bit base component, where inliers are captured faithfully while outlier are inevitably quantized. A sparse 4-bit residual component is then introduced to compensate for these quantization errors, selectively targeting the most error-critical weight blocks where output distortion is shown to be concentrated. However, a unified representation alone is insufficient, as naïvely executing the sparse residual as a separate kernel still breaks the unified low-bit inference pipeline. To bridge this gap, we introduce \textbf{ZipperEngine}, which fuses sparse block computation into the dense 4-bit GEMM kernel via an overlapped pipeline, unifying not only the representation but also the execution into a single coherent low-bit inference pipeline. Extensive experiments on LLaMA3 and Qwen3 demonstrate that MosaicQuant preserves near-FP16 accuracy while achieving up to $1.24\times$ speedup over the W16A16 baseline.

*   **现有方法的局限：** 现有混合精度方法（如Atom， ResQ）虽能通过将“离群点”保持在更高精度（如8-bit）来恢复精度，但引入了非统一的执行路径。这需要额外的格式转换、数据搬运和分派不同精度的内核核，破坏了低位计算的高效性和规律性，无法在实际硬件上实现预期的加速。
*   **研究挑战：** 如何在保持完全统一的4-bit表示和执行的前提下，有效补偿离群点带来的量化误差，同时不引入过高的计算和延迟开销。
*   **整体含义：** 本文通过提出一种全新的内点-离群点分解（inlier-outlier disaggregation）思想，首次实现了既能保持接近FP16精度，又能通过统一4-bit流水线获得实际推理加速的量化范式，为LLM的高效部署提供了新途径。

## 2. 方法论

*   **核心思想：** MosaicQuant提出将权重矩阵分解为两部分：
    1.  **密集的4-bit基分量（Dense 4-bit Base Component）：** 用于忠实表示绝大多数的“内点”权重。
    2.  **稀疏的4-bit残差分量（Sparse 4-bit Residual Component）：** 用于补偿因“离群点”量化而引入的误差。
    这种分解确保了所有权重操作都在4-bit的统一表示下进行，实现了精度与效率的理想平衡。

*   **关键技术细节：**
    *   **1. 块级（Block-level）补偿的有效性：** 论文发现，离群点引起的输出失真并非集中在整个通道，而是呈现细粒度的块级集中模式。因此，残差补偿应该在块级别进行，比通道级别更精确，可以避免补偿大量无关值。
    *   **2. Hessian引导的残差块选择：**
        *   **问题：** 如何高效地选择哪些权重块需要稀疏残差补偿？
        *   **方案：** 提出一个激活感知的 `∆-Hessian` 分数。该分数基于输入激活的Hessian矩阵（对角近似）和量化次差，评估补偿某个块对最终输出误差的预期减少量。它综合了权重大小和激活敏感性。
        *   **算法流程：**
            1.  对权重进行4-bit量化，得到残差 `R = W_hat - Q(W_hat)`。
            2.  计算激活灵敏度 `h_j`（基于输入激活的平方和）。
            3.  对每个候选块计算其 `∆-Hessian` 分数 `SB`。
            4.  根据稀疏预算，选择Top-K个分数最高的块进行残差补偿。
            5.  最终量化权重为：`WMQ = Q(W_hat) + Mask * Q(R)`。
    *   **3. ZipperEngine 引擎：**
        *   **问题：** 稀疏残差计算如果作为独立内核执行，会引入额外内核启动和数据搬运开销，抵消4-bit推理的优势。
        *   **方案：** ZipperEngine 通过重叠流水线（overlapped pipeline）技术，将稀疏块的计算融合到密集4-bit GEMM内核中。它利用图块化结构，将密集矩阵乘法、稀疏残差计算、归约求和等步骤调度在一个连贯的内核中，隐藏了内存访问和辅助操作的延迟，实现了计算与数据搬运的高度重叠。

*   **对校准数据的依赖：** 像所有PTQ方法一样，MosaicQuant的性能很大程度上依赖于校准数据集的质量和代表性。如果部署域与校准域差异很大，性能可能会下降。
*   **硬件依赖性：** ZipperEngine的设计针对特定的GPU架构（如NVIDIA的Tensor Core和CUDA Core）。向其他平台（如AMD GPU、手机芯片、ASIC）的迁移可能需要重新设计或适配。
