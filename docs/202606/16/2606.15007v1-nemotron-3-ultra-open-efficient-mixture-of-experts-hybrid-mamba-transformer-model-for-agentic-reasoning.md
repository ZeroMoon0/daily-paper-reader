## 摘要
我们推出了Nemotron 3 Ultra，一个总参数量5500亿、激活参数550亿的混合专家（MoE）混合Mamba-注意力语言模型。我们在20万亿文本token上预训练了Nemotron 3 Ultra，随后将上下文长度扩展至100万token，并使用监督微调（SFT）、强化学习（RL）和多教师在线策略蒸馏（MOPD）进行后训练。Nemotron 3 Ultra是我们迄今为止能力最强的模型，采用了多项关键技术——LatentMoE、多token预测（MTP）、NVFP4预训练、多环境RLVR、MOPD以及推理预算控制。与最先进的公开大语言模型相比，Nemotron 3 Ultra在保持同等精度的同时，推理吞吐量提升了约6倍。最先进的精度、高推理吞吐量和100万token的上下文长度使Nemotron 3 Ultra成为长时间运行的自主智能体任务的理想选择。我们在HuggingFace上开源了基础、后训练和量化模型检查点，以及训练数据和配方。

## Abstract
We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. We pre-trained Nemotron 3 Ultra on 20 trillion text tokens, then extended the context length to 1M tokens, and post-trained using Supervised Fine Tuning (SFT), Reinforcement Learning (RL), and Multi-teacher On-Policy Distillation (MOPD). Nemotron 3 Ultra is our most capable model yet, employing multiple key technologies - LatentMoE, Multi Token Prediction (MTP), NVFP4 pre-training, multi-environment RLVR, MOPD, and reasoning budget control. Nemotron 3 Ultra achieves up to ~6x higher inference throughput as compared to state-of-the-art publicly available LLMs while attaining on-par accuracy. The state-of-the-art accuracy, high inference throughput, and 1M token context length make Nemotron 3 Ultra ideal for long-running autonomous agentic tasks. We open-source the base, post-trained, and quantized checkpoints, along with the training data and recipe on HuggingFace.

### 2.1 核心思想

### 2.2 关键技术细节

- **架构设计**：

- **预训练**：
  - **数据**：20万亿token，分为两阶段：第一阶段（15万亿）侧重数据多样性，第二阶段（5万亿）侧重数据质量。包含网页语料、数学、代码、学术、法律、多语言等，并使用了新的合成数据集（如Code refresh、Legal、Fact-Seeking等）。
  - **NVFP4精度预训练**：权重使用E2M1格式，二维块量化，输入使用随机Hadamard变换，梯度使用随机舍入。在如此大规模下首次证明NVFP4训练的稳定性（与BF16训练损失差距<0.4%）。
  - **长上下文扩展**：在预训练最后阶段进行连续预训练（CPT）至100万token，使用1M和4K序列长度混合训练，以保持短任务性能。

- **后训练管道**：
  - **SFT**：两阶段训练，涵盖长上下文、推理效率与控制、安全、搜索、终端使用、数学/代码、科学、对话、多语言等任务。
  - **RLVR（带可验证奖励的强化学习）**：在一个统一的异步GRPO框架下，针对多个环境（终端、搜索、软件工程等）进行训练，使用8K全局批次、最大生成长度64K token。
  - **MOPD（多教师在线策略蒸馏）**：训练了10个以上的专用教师模型（如软件工程、办公、搜索、STEM、竞赛编程等），对学生生成的轨迹提供密集的token级奖励信号。采用异步PPO风格优化，包含行为策略与近端策略的解耦及重要性采样。该过程进行了两轮迭代，教师能力持续演化。
  - **MTP Boosting**：在MOPD之后冻结主干，仅更新MTP头，使用学生生成轨迹的前向KL蒸馏，解决训练-推理分布不匹配问题。

- **推理预算控制**：支持“中等努力”模式，通过截断推理痕迹并屏蔽`</think>` token损失来实现，可灵活调整准确率-效率曲线。

### 2.3 核心公式/算法说明（文字描述）

- **MOPD损失函数**：最大化学生策略与教师策略的负反向KL散度。实际使用clip后的PPO风格代理目标，且包含了行为到近端策略的重要性权重，以处理异步环境下的策略陈旧问题。
- **MTP Boosting损失**：对MTP步骤k，计算学生MTP头logits与主干logits的KL散度，使用温度T=2缩放，并乘以T²因子，仅针对助手token进行计算。
- **NVFP4训练**：使用E2M1权重、二维块缩放，配合随机舍入和随机Hadamard变换，实现4-bit精度的稳定训练。

## 6. 结论与未来方向

- **核心结论**：
  - **Nemotron 3 Ultra**通过混合Mamba-Attention-MoE架构，以55B激活参数实现了与1T+参数Transformer模型相当的智能体能力，同时推理吞吐量提升2–6倍。
  - NVFP4精度预训练首次在550B规模上证明可行，为大规模低精度训练提供了经验证据。
  - 多教师在线策略蒸馏（MOPD）是提升智能体长期推理和工具使用能力的关键驱动力，尤其在需要多步骤拆解和回溯的场景中表现突出。
  - MTP Boosting作为轻量级后训练手段，显著提升了推测解码的接受长度，进一步降低推理延迟。

- **局限性**：
  - 尽管在英语智能体任务上表现优异，多语言能力（尤其非英语工具调用）仍有限，论文报告在多语言Agent基准上存在2–5%的性能差距。
  - 混合架构在NVFP4训练下的稳定性虽得到验证，但理论解释尚不充分，量化训练过程中的数值漂移风险仍需更严格的监控。
  - 后训练管道高度依赖高质量教师模型和合成数据，这增加了工程复杂度；教师模型的持续迭代是否可收敛尚不确定。

- **未来方向**：
  - 将混合架构扩展至更大规模（如1T总参数），并探索更高效的MoE路由策略（如Dense-MoE混合）。
  - 在NVFP4基础上进一步压缩至NVFP2或混合精度，同时保持训练稳定性。
  - 构建完全自适应的推理预算控制系统，使模型能够根据任务复杂度动态决定“思考”深度。
  - 开源全部训练配方、数据合成脚本及推理优化工具链，以推动社区在高效Agent模型上的研究。
