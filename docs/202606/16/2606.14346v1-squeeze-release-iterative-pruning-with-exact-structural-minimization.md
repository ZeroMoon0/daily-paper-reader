## 摘要
非结构化剪枝会产生稀疏权重张量，但标准实现保持张量形状不变，因此部署后的模型并不比剪枝前更小。我们提出了一种精确的结构重写，称为最小化，它将掩码网络转换为一个更小的稠密网络，其前向函数在浮点舍入误差内相同。挤压-释放循环迭代进行剪枝和最小化，并加入一个中间释放步骤，将压缩张量内的精确零位置重新启用为小校准噪声，从而将原本浪费的容量转化为可训练参数。连续的循环利用该容量找到单次剪枝无法触及的结构冗余。我们还引入了CompensatedLayerNorm，这是一种保持功能的LayerNorm替代方案，将最小化扩展到跨LayerNorm残差流的通道缩减。在全连接模型网络上，挤压-释放将可部署网络压缩至未剪枝模型的39倍，在现代CNN（ConvNeXt-Tiny）上压缩至14.8倍，且精度相当。此外，我们证明了该重写可以扩展到Transformer架构。

## Abstract
Unstructured pruning produces sparse weight tensors, but the standard implementation keeps tensor shapes unchanged so the deployed model is no smaller than before pruning. We present an exact structural rewrite, which we call minimization, that converts a masked network into a smaller dense network with the same forward function up to floating-point rounding. The Squeeze-Release cycle iterates pruning and minimization with an intermediate release step that re-enables the exact-zero positions inside the compacted tensors as small calibrated noise, turning otherwise wasted capacity back into trainable parameters. Successive cycles use that capacity to find structural redundancy a single pass cannot reach. We additionally introduce CompensatedLayerNorm, a function-preserving replacement for LayerNorm that extends minimization to channel reduction across LayerNorm-equipped residual streams. Squeeze-Release compresses the deployable network to 39x smaller than the unpruned model on a fully-connected model network and 14.8x smaller on modern CNN (ConvNeXt-Tiny), at comparable accuracy. In addition we prove that the rewrite can be extended to transformer architectures.

# 论文总结：Squeeze-Release: Iterative Pruning with Exact Structural Minimization

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：非结构化剪枝虽能产生高稀疏度的权重张量，但标准实现（如 PyTorch 的 `torch.nn.utils.prune`）**不改变张量形状**，仅通过掩码在前向时乘以零。因此：
  - 模型在磁盘/内存中的大小与未剪枝时相同；
  - 前向传播仍运行完整的稠密计算核；
  - 广泛报告的“掩码存活参数数量”（mask-alive count）**不等于实际可部署模型的大小**，在通用硬件上无法获得显著的加速或内存节省。
- **核心矛盾**：剪枝包含两个独立任务——决定**哪些参数移除**与**实际移除参数**。前者是重要性评分和训练策略问题（大量文献），后者是**结构重写问题**：如何将带已知稀疏模式的网络转化为更小的稠密网络且保持前向函数不变。
- **本文目标**：提出一种精确的结构重写（称为**最小化 Minimization**）作为后处理步骤，并构建迭代循环（**挤压-释放 Squeeze-Release**）来逐步缩小可部署模型，同时引入对 LayerNorm 的兼容支持。

## 2. 方法论

### 核心思想
- **最小化**：将带掩码的网络精确转化为更小的稠密网络（输出差异仅为浮点舍入误差）。通过识别并移除**全零行/全零列/全零滤波器**（dead-incoming/outgoing units），并将前向偏置折叠到下游层。
- **CompensatedLayerNorm**：当 LayerNorm 的输入通道被减少时，标准 LayerNorm 会因为统计量变化而改变输出。CompensatedLayerNorm 存储被移除通道的三个标量统计量（计数 K、和 S、平方和 Q），在每次前向时从保留通道的统计量中**精确重建完整通道的均值和方差**，从而保持函数不变。额外存储为每个 LayerNorm 仅 3 个标量。
- **挤压-释放循环**：
  1. **Prune**：使用梯度×权重幅度的评分函数对权重进行非结构化或过滤器级剪枝（Conv2d 按滤波器粒度，Linear 按标量粒度），采用立方调度逐步提高稀疏度。若验证精度下降过快则回滚。
  2. **Squeeze**：应用精确最小化，移除全零神经元/通道/滤波器，压缩网络结构。
  3. **Release**：在压缩后稠密张量中**原本为零的位置**注入小的高斯噪声（标准差为存活权重的 0.01），将这些原本浪费的参数恢复为可训练。
  4. **Fine-tune**：对释放后的网络进行微调。
- 循环重复直到无法再缩小或达到最大周期数。

### 关键技术细节
- **全零行（dead-incoming）**：若某神经元（或卷积输出通道）的所有输入权重为零，其输出为常数。可将该常数乘以后续层对应权重再加到后续层偏置中，然后删除该行/列。
- **全零列（dead-outgoing）**：若某下游层对应列全部为零，可直接删除上游该行和下游该列。
- **ConvNeXt 块的三种缩减模式**：
  - Mode A：缩减倒置瓶颈内部 4C 维度（点卷积之间）。
  - Mode B：缩减残差流通道（DWConv 到 PWConv1 之间），需使用 CompensatedLayerNorm。
  - Mode C：移除整个块（当所有 DWConv 滤波器为零时，将块输出作为常数折叠到上游偏置）。
- **释放的动机**：最小化只移除全行/列，但零可能存在于非全零的行/列中，这些“浪费的容量”不会贡献输出但占用计算。释放使其重新成为可训练参数，后续剪枝周期可利用它们找到新的结构冗余。

## 3. 实验设计

### 数据集与模型
- **全连接网络 (FC) on MNIST**：结构 [784,128,256,128,128,64,10]，含 BatchNorm 和 SELU，约 193k 参数（191k 可剪枝）。训练 160 轮基线，准确率 98.24%。
- **ConvNeXt-Tiny on CIFAR-10**：从公开预训练 checkpoint 开始（27.8M 参数），输入 224x224，增强使用随机裁剪和水平翻转。训练/验证划分 45k/5k。
- **ViT-Small on ImageNet-1k**（概念证明）：timm 预训练 vit_small_patch16_224（22M 参数），12 个块，384 维嵌入。仅运行 No-minimize 基线 + 后处理最小化，一次种子。

### 对比方法
- **No-minimize 基线**：仅进行剪枝和微调，不执行挤压和释放步骤，掩码跨周期累积。最终后处理应用最小化得到可部署大小。
- **Squeeze-Release Mode A**：仅使用 Mode A 缩减（和 Mode C）。
- **Squeeze-Release Mode AB**：同时使用 Mode A + Mode B（含 CompensatedLayerNorm）。

### 评估指标
- 测试准确率（%）
- 掩码存活参数数（mask-alive count）
- 可部署参数量（deployable count，即最小化后的稠密网络大小）
- 周期数（cycles）

## 6. 主要结论与发现

1. **精确最小化成立**：对任何非结构化剪枝器输出的掩码网络应用最小化，可得到更小的稠密网络，其前向函数在浮点精度内一致。
2. **Squeeze-Release 显著缩小可部署模型**：
   - FC/MNIST：可部署参数从基线后处理的 18.1k 进一步降至 4.9k（39.2× 相对于未剪枝模型），精度无统计显著差异。
   - ConvNeXt-Tiny/CIFAR-10：后处理的基线可部署 10.0M，Mode A 降至 2.45M（11.4×），Mode AB 降至 1.88M（14.8×），精度相当（90.27% vs 90.81%）。
3. **CompensatedLayerNorm 带来额外压缩**：Mode AB 比 Mode A 额外缩小 1.3×，且保持功能精确（最大绝对 logit 差异 1.06×10⁻⁶）。
4. **释放步骤的关键作用**：No-minimize 平均 3.6 周期终止（因为无法有效复用零容量），而 Squeeze-Release 可运行 97-100 周期，说明每次释放确实将废容量转换为可训练参数，使后续剪枝能发现新的结构冗余。
5. **掩码存活数与可部署数之间的巨大差距**：在 FC 上 5.4×，ConvNeXt 上 11.3×，ViT 上 5.0×。报告后者才反映实际部署成本。
6. **最小化可扩展至 Transformer**：对 ViT-Small 的后处理最小化（移除全零注意力头、MLP 缩小等）成功将 22M 模型缩小 12%，且前向函数一致。

## 7. 优点

- **方法新颖且实用**：解决了非结构化剪枝长期被忽视的“部署体积不减”问题，提供即插即用的后处理工具。
- **精确性保证**：最小化和 CompensatedLayerNorm 均保持前向函数精确（浮点舍入误差内），无需近似或长程微调恢复。
- **通用性**：可应用于任意非结构化掩码（与重要性评分方法无关），并扩展到 CNN 和 Transformer 架构。
- **释放机制设计精巧**：意识到最小化后仍存在“浪费容量”，并迭代恢复这些位置，形成正反馈循环。
- **实验严谨性较好**：多种子、统计检验、不同模式消融、并报告 mask-alive 与 deployable 的差距。

## 8. 不足与局限

- **实验覆盖有限**：主要在小规模数据集（MNIST、CIFAR-10）上验证，ImageNet-1k 仅用于概念证明且训练不充分（40%数据子集、低精度阈值）。缺乏在大规模模型（如 LLM、更大 ViT）上的系统评估。
- **与现有方法的对比不足**：未与广泛使用的结构化剪枝（如 Network Slimming、ThiNet）、知识蒸馏（Minitron）、或近似剪枝 (SliceGPT、Pangu Light) 进行公平比较。
- **计算开销大**：迭代循环需要多次训练（ConvNeXt 上有时多达 100 个周期），wall-clock 开销远高于单次剪枝+微调，虽在一次性压缩场景下可接受，但限制了实用性。
- **释放初始化选择未深入优化**：作者指出将零位置重新初始化为零也能得到类似结果，表明释放的具体分布可能不关键，但未探索更优的初始化策略。
- **ViT 实验仅单种子且未运行完整循环**，不能证明迭代循环对 Transformer 的有效性，仅说明后处理最小化可行。
- **对实际加速的讨论不足**：虽然缩小了参数量，但未测量实际推理延迟或内存占用变化，也未讨论压缩后的模型是否在标准硬件上获得与参数缩减成比例的加速。
- **与修剪调度之间的交互未充分分析**：如如何选择目标稀疏度、周期数、提前终止条件等超参数对最终结果的影响未进行消融。
