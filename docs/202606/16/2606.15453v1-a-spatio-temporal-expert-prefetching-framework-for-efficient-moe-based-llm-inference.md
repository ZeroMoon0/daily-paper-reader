## 摘要
基于混合专家模型的大语言模型，如Qwen和DeepSeek，近年来已成为提升模型容量而不成比例增加计算成本的有效方法。通过将密集大语言模型中的传统前馈网络替换为一组专家，并对每个输入标记仅激活其中一部分，MoE模型显著增加了总参数数量，同时保持每标记计算相对可控。然而，这种动态且不规则的专家激活模式在推理过程中也引入了大量专家加载开销，因为所需专家必须根据依赖于标记的路由结果按需获取。因此，专家加载延迟成为性能和能效低下的主要来源。为此，我们首先对多种基于MoE的大语言模型及其应用（包括语言理解和代码生成）中的专家选择行为进行了全面分析。分析揭示，在每个应用领域内，专家请求在相邻MoE层和连续解码标记之间均表现出强相关性，使得未来专家激活可预测。基于这一洞察，我们提出ST-MoE，一种时空专家预取框架，该框架在专家使用前主动预取，使专家加载与正在进行的计算重叠。ST-MoE结合了轻量级运行时预测机制（保留原始路由行为）与可重构硬件设计（高效支持动态专家预取）。预测机制与支撑硬件的协同作用显著提升了MoE推理性能和能效，同时保持了模型推理精度。

## Abstract
Mixture-of-Experts (MoE) based large language models (LLMs), such as Qwen and DeepSeek, have recently emerged as an effective approach to improving model capacity without proportionally increasing computational cost. By replacing the conventional feed-forward network in dense LLMs with a set of experts and activating only a subset of them for each input token, MoE models significantly increase the total number of parameters while keeping the per-token computation relatively manageable. However, this dynamic and irregular expert activation pattern also introduces substantial expert loading overhead during inference, since the required experts must be fetched on demand according to token-dependent routing results. As a result, expert loading latency becomes a major source of performance and energy inefficiency. To this end, we first perform a comprehensive analysis of expert selection behavior in various MoE-based LLMs and applications, including language understanding and code generation. Our analysis reveals that, within each application domain, expert requests exhibit strong correlation across both adjacent MoE layers and consecutive decoding tokens, making future expert activations predictable. Based on this insight, we propose ST-MoE, a spatio-temporal expert prefetching framework that proactively stages experts ahead of use to overlap expert loading with ongoing computation. ST-MoE combines a lightweight runtime prediction mechanism that preserves the original routing behavior with a reconfigurable hardware design that efficiently supports dynamic expert prefetching. The combined effect of the prediction mechanism with the supporting hardware significantly improves MoE inference performance and energy efficiency while preserving model inference accuracy.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：大型语言模型（LLM）通过混合专家（MoE）技术在不显著增加每 token 计算成本的情况下大幅提升模型容量（如 Qwen、DeepSeek）。但 MoE 的推理过程面临一个关键瓶颈：每个 token 仅激活部分专家，且激活模式高度依赖于输入，导致专家权重必须按需从片外内存加载，专家加载延迟成为性能和能效的主要障碍。
- **核心问题**：如何在不改变原始路由行为、不引入额外训练开销的前提下，有效隐藏专家加载延迟，提升 MoE 推理效率。
- **整体意义**：该工作首次系统记录并利用了 MoE 推理中跨层（空间）和连续 token 间（时间）的专家选择相关性，提出轻量级运行时预测与可重构硬件协同的专家预取框架（ST-MoE），实现了显著的性能提升和能效改善，同时保持推理精度不变。

## 2. 方法论

### 2.1 核心思想

通过分析发现，MoE 专家选择在相邻层（空间）和连续解码 token（时间）上存在强相关性。利用这种可预测性，ST-MoE 在下一层专家需求产生前，基于当前层的路由结果和历史信息，轻量级地预测未来专家，并提前从片外内存预取到片上 buffer，使专家加载与当前层计算（注意力块等）重叠，从而隐藏加载延迟。

### 2.2 关键技术细节

- **数据结构**：
  - **Cross-layer Correlation Table (CCT)**：记录当前层每个专家与下一层 Top-K 相关专家，含 2-bit 置信度（10=preferred, 11=strongly preferred, 01=not preferred, 00=strongly not preferred）。通过 profiling 少量 token 构建，运行时动态更新。
  - **History Table (HT)**：记录每层上一个 token 实际激活的 Top-K 专家，置信度固定为 10，每 token 后覆盖更新。

- **预测算法（Algorithm 2 - Expert Predict）**：
  - 当前层专家激活集 \(E_i\) 索引 CCT，累加候选专家的置信度分数。
  - HT 中每个专家加 1 分；若同时出现在 CCT 和 HT 中，则分数相加（式 1）。
  - 所有分数 ≥ 2（对应 10 状态）的专家作为预测集 PrefetchSet，发送给硬件预取。

- **表更新算法（Algorithm 3 - Table Update）**：
  - 实际路由结果 \(F_{i+1}\) 到达后，更新 CCT 对应条目：
    - 若预取专家命中，置信度加 1（上限 3）；
    - 若未命中且置信度 > 0，减 1；
    - 若置信度降至 0，用未在 PrefetchSet 中的实际专家替换，置信度重置为 10。
  - HT 直接替换为最新路由结果。

- **可重构硬件架构**：
  - **Expert Prediction Unit (EPU)**：存储 CCT/HT，执行轻量预测和表更新。
  - **Expert Mapping Unit (EMU)**：根据实际路由结果将专家映射到处理单元（PE），并配置置换网络路由数据和权重。
  - **Processing Elements (PEs)**：每个 PE 含 64×64 MAC 阵列，支持输入静止或权重静止数据流，由局部控制器根据专家计算特点重新配置。
  - **流水线执行**：专家预取与注意力计算重叠；预测和表更新与当前层计算并行；预测错误的专家即时补取，不影响正确性。
