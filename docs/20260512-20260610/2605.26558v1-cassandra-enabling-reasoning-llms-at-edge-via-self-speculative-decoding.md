## 摘要
推测解码已成为加速大型语言模型（LLM）的一种有前景的无损方法。随着推理型LLM日益面临解码阶段开销并受基于近似的方法降低准确性的问题，无损推测解码对于高效推理变得至关重要。然而，现有方法仍然难以在无需额外训练的情况下提供强大的低批量性能，限制了在消费设备上的实际部署。为应对这一挑战，我们提出了Cassandra，一种算法与硬件协同设计的自推测解码框架，专为低批量场景优化。Cassandra通过细粒度数据选择构建了一个高性能、无需训练的草稿模型。利用优化的剪枝和尾数截断，它在模型权重和键值（KV）缓存中识别出最显著的值，从而在全精度并行验证之前实现快速的候选令牌生成。与先前基于层跳过或结构化KV压缩的自推测解码方法不同，Cassandra实现了显著更高的效率。为进一步减少Cassandra表示与标准浮点格式之间的格式转换开销，我们还引入了一个轻量级的编码器-解码器硬件模块，旨在与商业GPU和NPU无缝集成。实验结果表明，Cassandra在无需额外训练的情况下，相比BF16基线实现了高达2.41倍的加速。此外，在运行于NVIDIA GeForce RTX 4090上的Llama 3 8B上，在相同内存预算下，Cassandra相比最先进的推测解码方法Eagle-3生成了1.81倍的令牌。

## Abstract
Speculative decoding has emerged as a promising lossless approach for accelerating Large Language Models (LLMs). As reasoning LLMs increasingly suffer from decode-stage overhead and approximation-based methods degrade accuracy, lossless speculative decoding has become essential for efficient inference. However, existing methods still struggle to deliver strong low-batch performance without additional training, limiting practical deployment on consumer devices. To address this challenge, we propose Cassandra, an algorithm-hardware co-designed self-speculative decoding framework optimized for low-batch scenarios. Cassandra constructs a high-performance, training-free draft model through fine-grained data selection. Using optimized pruning and mantissa truncation, it identifies the most salient values in both model weights and the Key-Value (KV) cache, enabling rapid candidate token generation before full-precision parallel verification. Unlike prior self-speculative decoding methods based on layer skipping or structured KV compression, Cassandra achieves significantly higher efficiency. To further reduce the overhead of format conversion between Cassandra representations and standard floating-point formats, we also introduce a lightweight encoder-decoder hardware module designed for seamless integration with commercial GPUs and NPUs. Experimental results show that Cassandra achieves up to 2.41x speedup over the BF16 baseline without additional training. Furthermore, on Llama 3 8B running on an NVIDIA GeForce RTX 4090, Cassandra generates 1.81x more tokens under the same memory budget compared to Eagle-3, a state-of-the-art speculative decoding method.

#### 2. 方法论：核心思想与技术细节
Cassandra的核心思想是：**直接从目标大语言模型本身，通过细粒度的数据选择，构建一个高质量的草稿模型，无需额外训练。**

**Cassandra算法流程分为三个阶段：**
*   **阶段一：初始格式转换**，将原始模型的权重和KV缓存分为两部分：
    *   **推测数据 (Speculation Data)**：用于草稿模型推理的数据。
    *   **验证数据 (Verification Data)**：在全精度并行验证时补充使用的数据。
*   **阶段二：草稿模型推理**：仅加载推测数据，快速生成候选令牌。
*   **阶段三：目标模型推理**：同时加载推测数据和验证数据，恢复完整模型表示，进行并行验证。

**关键技术细节：**
1.  **提出基于细粒度数据选择的草稿模型构建方法** (Fig. 4):
    *   **非结构化值剪枝 (Unstructured Value Pruning)**：
        *   **权重**：采用`Wanda`方法，通过校准数据计算激活值的L2范数，与权重逐元素相乘得到重要性分数，进行非结构化剪枝。
        *   **KV缓存**：采用`Mustafar`方法，逐令牌进行基于幅度（per-token magnitude）的剪枝，对Key Cache效果显著，对Value Cache也有效。
    *   **尾数截断 (Mantissa Truncation)**：减少每个值的位宽（保留高位比特），因其保留了原始比特的子集，保证了草稿模型是目标模型的“严格子集”，避免了额外的内存开销。
    *   **指数压缩 (Exponent Compression)**：针对BF16格式中指数位（占50%）的开销，提出两种方案：
        *   **Cassandra-1 (无损)**：基于**一元编码 (Unary Coding)**。通过分析发现模型权重和KV缓存的指数分布熵很低（约2.7 bits），一元编码可以无损压缩指数至平均2.85 bits。该方案解码逻辑简单，适合硬件并行。
        *   **Cassandra-2 (有损，更高压缩率)**：基于**MX格式 (Microscaling Format)**，让多个浮点数共享一个指数，以轻微精度损失换取更高压缩率。
2.  **设计轻量级硬件解码器与编码器** (Fig. 8):
    *   **解码器**：用于在线数据重建。针对一元编码和MX格式设计了不同的数据流，核心模块是**“并行零计数器 (Parallel Zero Counter)”**，能高效地解析一元编码流。
    *   **编码器**：主要用于KV缓存的在线格式转换。通过寻找**top-k**值将数据分为推测组和验证组，并对推测组的指数进行压缩。
3.  **硬件集成方案** (Fig. 10):
    *   解码器/编码器被集成到GPU的每个内存分区中的一个L2 Cache与互联网络之间，以及NPU的DMA控制器中。通过页表标识区分标准数据和Cassandra格式数据。
4.  **Superblock数据管理** (Fig. 11): 提出基于超级块的内存映射方案，将多种数据类型（位图、变长指数、尾数）打包成固定的超级块进行加载，并通过解码器内部缓冲区处理数据对齐问题，提升了内存访问效率。
