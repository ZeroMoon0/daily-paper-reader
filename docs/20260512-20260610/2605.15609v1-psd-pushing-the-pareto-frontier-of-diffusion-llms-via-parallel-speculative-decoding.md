## 摘要
扩散大语言模型通过迭代去噪掩码标记序列来生成文本。尽管扩散大语言模型可以在每一步中并行预测所有掩码位置，但大量的去噪迭代仍然使推理成本高昂。这种成本可以通过空间上每一步解除多个标记的掩码，或时间上将多个去噪步骤合并为一次验证调用来降低。我们提出并行投机解码，这是一个无需训练的框架，能够沿着两个轴同时改进推理。利用单次前向传播中的置信度分数，PSD通过可配置的自适应解掩码策略选择要解掩码的位置，并在无需额外模型调用的情况下构建多深度投机草稿。最终批处理验证步骤应用分层接受，保留与更新预测一致的最深草稿。在三个扩散大语言模型上进行的推理和代码生成任务实验表明，PSD在推理效率与生成质量之间实现了有利的权衡，每次前向传播可达5.5倍的标记数，且准确性与贪婪解码相当。

## Abstract
Diffusion large language models (dLLMs) generate text by iteratively denoising masked token sequences. Although dLLMs can predict all masked positions in parallel within each step, the large number of denoising iterations still makes inference expensive. This cost can be reduced spatially by unmasking multiple tokens per step, or temporally by collapsing multiple denoising steps into one verification call. We propose Parallel Speculative Decoding (PSD), a training-free framework that jointly improves inference along both axes. Using the confidence scores from a single forward pass, PSD selects positions to unmask via a configurable, adaptive unmasking policy and constructs multi-depth speculative drafts without extra model calls. A final batched verification pass then applies hierarchical acceptance, keeping the deepest draft that remains consistent with the updated predictions. Experiments on three dLLMs across reasoning and code generation tasks show that PSD achieves favorable trade-offs between inference efficiency and generation quality, reaching up to $5.5\times$ tokens per forward pass with accuracy comparable to greedy decoding.

## 1. 核心问题与整体含义（研究动机和背景）

扩散大语言模型（dLLMs）通过迭代去噪掩码序列生成文本，具有天然并行性（每步可并行预测所有掩码位置），但大量去噪步数（通常需要多步前向传播）导致推理成本高昂，成为部署瓶颈。现有加速方法分为两类：

- **空间加速**：每步解掩码多个token（如 Confidence、LocalLeap 等），但增加并行度会导致低置信token过早提交、错误传播，质量急剧下降。
- **时间加速**：投机解码（如 Spiffy）将多个去噪步合并为一次验证调用，但每步仍只解一个token，加速上限受接受率限制。

这两类方法各自存在质量‑速度帕累托前沿，无法同时大幅提升。论文发现空间和时间轴具有互补性：空间轴决定每步揭示多少token，时间轴决定将多少步折叠为一次验证，两者结合能获得远超单轴的加速效果。由此提出 **PSD（Parallel Speculative Decoding）**，一个无需训练的通用框架，统一优化两个维度。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
利用去噪过程中高置信度位置的稳定性：当前步识别的高置信位置在后续几步大概率保持不变。基于此，PSD 在单次前向传播后，用当前置信度排序构造多深度投机草稿，并通过层次批验证接受最长一致分支，从而同时实现空间并行和时间投机。

### 三阶段流程
**阶段1：空间并行解掩码（Spatial Parallel Unmasking）**
- 采用任意并行解码策略 π（如置信度阈值、LocalLeap）选择当前步要揭示的位置集合 T⁽ᵗ⁾，更新序列状态 x⁽ᵗ⁺¹⁾。
- 每步平均揭示 ¯r_π 个token，减少总迭代次数。

**阶段2：时间投机草稿构造（Temporal Speculative Drafting）**
- 在阶段1之后，利用缓存的上一步置信度分数，对剩余掩码位置按置信度降序排序得到 σ。
- 构建有向无环图 G，每个节点 k 对应一个子集 S_k ⊆ {σ₁,…,σ_m}，代表投机解掩码的位置集。根节点 S_r=∅（当前状态）。
- 节点 k 的草稿序列 ˜x_k 定义为：对 i∈S_k 填入缓存预测 ˆx_i⁽ᵗ⁾，其他位置保持 x⁽ᵗ⁺¹⁾。所有草稿通过简单的索引操作生成，无需额外模型调用。
- G 的结构离线校准以最大化经验吞吐量。

**阶段3：批验证与层次接受（Batched Verification & Hierarchical Acceptance）**
- 将所有草稿序列组成一个批次，送入模型进行一次前向传播，得到每个草稿每个位置的更新预测和置信度。
- 按拓扑序遍历 DAG：根节点无条件接受；子节点 k 在存在至少一个已接受的父节点 p 且所有新投机位置 i∈S_k\S_p 满足接受准则（与空间策略 π 一致）时被接受。
- 选择接受集合中 |S_k| 最大的节点作为当前步的最终状态 x⁽ᵗ⁺¹⁾，并额外提交那些通过接受条件的验证器预测。

**算法概览（Algorithm 1）**：循环直到块内所有掩码位置被填满。每次循环包括一次空间前向传播（阶段1）、草稿构造（阶段2）、一次批验证前向传播（阶段3）。总体每步只需两次前向传播（一次空间、一次批验证），但可揭示大量token。

## 6. 主要结论与发现

1. **空间与时间加速互补**：PSD 通过联合优化两个轴，获得的加速远超任一单轴方法。例如 LLaDA-1.5 在 GSM8K 上，PSD（Confidence）将 Spiffy 的 2.0× TPF 提升到 4.0×，额外增加 2.0× 空间加速，且质量几乎不变（78.9% vs 78.8%）。
2. **投机验证保持质量**：相比空间并行方法（如 LoPA k=5 在 HumanEval 上使 pass@1 从 35.4% 暴跌至 23.8%），PSD 在相近 TPF 下保持零退化（35.4%）。层次接受机制有效过滤错误投机，防止错误传播。
3. **模型无关性与策略灵活性**：在三个不同训练范式的模型上均优于基线，且 PSD 可以即插即用不同空间策略（Confidence 更保质量，LocalLeap 更高效）。
4. **代码生成对空间并行更敏感**：数学模型能吸收 token 扰动，但代码因语法严格，空间并行更容易导致失败。PSD 的投机机制缩小了这种差距。
5. **投机深度饱和**：d=3 已获得大部分加速收益（d>3 边际收益递减），且增加深度不显著降低质量，说明层次接受稳健。
6. **预测稳定性支撑**：当前高置信位置在未来步中仍大概率被解掩码（Precision@K 在短视距内很高），支撑了基于当前置信度构造草稿的合理性。
7. **阶段互补**：在块的不同进度中，并行和投机轮流主导（中间段并行为主，结尾段投机为主），说明两者自然协同。

## 7. 优点

- **无需训练**：PSD 完全无需额外训练或微调，兼容现有任意预训练 dLLM。
- **政策无关**：空间阶段可作为插件替换任意并行解码策略，具有广泛适用性和未来扩展性。
- **联合优化双轴**：首次在 dLLM 推理中同时利用空间并行和时间投机，获得乘积级加速。
- **层次接受机制**：通过 DAG 多父节点支持和条件接受，既防止错误传播，又能利用多个路径接受更深草稿。
- **草稿构造零开销**：草稿仅需索引操作和缓存预测，无需额外模型前向传播。
- **KV缓存共享**：批验证阶段可共享前缀 KV 缓存，减少冗余计算。
- **实验充分**：涵盖模型多样性、任务多样性、基线多样性，并附加机理分析，结论可靠。
