## 摘要
投机解码通过让轻量级草稿模型提出候选令牌的投机窗口，由更大的目标模型并行验证，从而加速LLM推理。在实践中，投机效率常受限于难以草稿的位置，早期不匹配会截断已接受的前缀并使投机窗口的其余部分失效。大多数基于学习的草稿模型仍然使用令牌级监督目标进行优化，尽管投机效用本质上是窗口级且对前缀敏感的。我们提出PPOW（性能驱动的自适应窗口策略优化），这是一种强化学习框架，将草稿优化从令牌级模仿转变为窗口级优化。PPOW结合了成本感知加速奖励、基于分布的邻近奖励以及自适应散度感知窗口，该窗口优先选择具有高置信度加权草稿-目标散度的信息窗口。在统一解码协议下，PPOW在多个模型系列和基准测试中实现了平均接受长度为6.29-6.52，加速比为3.39-4.36倍。这些结果表明，性能驱动的窗口级优化是提高投机解码效率的实用方法。

## Abstract
Speculative decoding accelerates LLM inference by having a lightweight draft model propose speculative windows of candidate tokens for parallel verification by a larger target model. In practice, speculative efficiency is often bottlenecked by hard-to-draft positions, where an early mismatch truncates the accepted prefix and invalidates the rest of the speculative window. Most learning-based drafters are still optimized with token-level supervised objectives, even though speculative utility is inherently window-level and prefix-sensitive. We propose PPOW (Performance-Driven Policy Optimization with Adaptive Windowing), a reinforcement learning framework that shifts drafter optimization from token-level imitation to window-level optimization. PPOW combines a Cost-Aware Speedup Reward, a Distribution-Based Proximity Reward, and Adaptive Divergence-Aware Windowing, which prioritizes informative windows with high confidence-weighted draft-target divergence. PPOW achieves average acceptance lengths of 6.29-6.52 and speedups of 3.39-4.36$\times$ across multiple model families and benchmarks under a unified decoding protocol. These results show that performance-driven window-level optimization is a practical approach to improving speculative decoding efficiency.

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：投机解码（Speculative Decoding）通过轻量级草稿模型（drafter）生成候选 token 的投机窗口，由大型目标模型并行验证，以加速 LLM 推理。实际中，若窗口内早期 token 被拒绝，整个窗口的后续 token 失效，形成**接受瓶颈**，严重限制加速效果。
- **现有局限**：当前主流学习型草稿器（如 EAGLE-3、GRIFFIN）仍使用 token 级监督（交叉熵）训练，优化目标为局部预测精度，但**投机效用在本质上是窗口级的、对前缀敏感的**。token 级提升并不直接转化为更长的接受前缀或更高效率。
- **核心问题**：如何设计训练目标，使草稿模型直接优化窗口级接受行为，从而突破接受瓶颈。
- **论文意义**：提出 PPOW 框架，将草稿优化从 token 级模仿转向**窗口级性能导向的强化学习**，显著提升平均接受长度和端到端加速比，为投机解码训练提供了新范式。

## 2. 方法论：核心思想、技术细节与算法
- **核心思想**：将投机窗口视为一个整体训练单元，使用强化学习（RL）直接优化窗口级效用，并采用自适应窗口选择聚焦于困难/关键位置。
- **关键技术细节**：
  - **窗口级策略优化**：对每个输入前缀，草稿策略 `πθ` 生成 `G_roll` 个投机窗口 `ŷ`，计算每个窗口的标量奖励，并在组内归一化得到组相对优势 `Â`。使用 PPO 风格的裁剪目标 + 目标锚定 KL 正则化（约束 `πθ` 接近目标模型 `π_target`）。
  - **成本感知加速奖励（Cost-Aware Speedup Reward）**：
    ```
    R_speedup = k / (kγ + 1)，其中 k 为接受长度，γ 为草稿模型相对目标模型的计算成本。
    ```
    该奖励平衡接受长度与草稿开销，比原始 k 或实测加速更稳定通用。
  - **基于分布的邻近奖励（Distribution-Based Proximity Reward）**：当 `k=0`（验证提前截断）时，若草稿窗口的累积目标对数似然与目标偏好窗口的差距小于阈值 `ϵ`，则给予额外奖励 `R_dist = η·1[Δ < ϵ]`。提供稀疏环境下的辅助信号。
  - **自适应散度感知窗口（Adaptive Divergence-Aware Windowing, ADAW）**：定义 token 级关键性得分 `v_t = C(P_t)·D_KL(P_t || Q_t)`，其中 `C(P_t) = 1 - H(P_t)/log|V|` 为置信度权重（低熵目标分布赋予更高权重）。窗口级得分 `s_j = (1/K) Σ v_t`，按 `s_j` 概率采样窗口，优先训练对接受瓶颈影响更大的窗口。
- **算法流程**（文字说明）：
  1. 对每个训练前缀，计算目标模型输出和草稿模型分布；
  2. 计算所有位置的 token 级关键性得分 `v_t`；
  3. 按 ADAW 采样一个起始窗口；
  4. 从该位置开始，草稿模型生成 `G_roll` 个投机窗口，经目标模型并行验证得到接受长度 `k`；
  5. 计算每个窗口的奖励（成本感知 + 邻近奖励），并归一化为组相对优势；
  6. 使用 PPO 目标（裁剪 + KL 正则化）更新草稿模型参数。

- PPOW 在所有设置下均优于 EAGLE-3 和 GRIFFIN，平均接受长度 6.29~6.52，加速比 3.39~4.36×（温度 0.0）。在 Qwen3-8B 上提升尤为显著（τ 从 6.16 到 6.52）。

- **窗口级 RL 优于继续监督训练**：相同额外训练步数下，CST 后期 τ 退化，而 PPOW 持续提升。
- **组件贡献**：消融实验证实 R_dist 和 ADAW 均有正贡献，二者结合效果最佳。ADAW 通过聚焦硬窗口加速了训练（图 4）。

## 7. 优点
- **问题洞察深刻**：明确指出现有训练的 token 级目标与窗口级接受行为之间的根本性不匹配，并设计相应解决方案。
- **奖励函数设计巧妙**：成本感知奖励权衡草稿与验证开销，避免依赖系统实测；邻近奖励在稀疏环境下提供辅助信号；两者结合使得训练信号更密集。
- **自适应窗口选择（ADAW）**：基于置信度加权的 KL 散度，理论上有根据（Pinsker 不等式 + 熵加权），实践上提升训练效率。
- **实验严谨**：统一解码协议，对比多种基线（含近期方法 OSD/Lookahead/FastDraft），消融充分，提供多角度分析（训练动态、窗口难易、候选大小）。
- **资源开销可接受**：8B 模型仅需约 50 GPU-hours，且开源自监督初始化。

## 8. 不足与局限
- **训练环境依赖**：RL 训练需要同时运行草稿和目标模型、执行多组滚动和验证，复杂度高于监督训练，且超参数较多（K、γ、η、ϵ、硬窗口 curriculum 等），调优成本高。

- **硬件依赖**：训练需大量 GPU 显存（因冻结目标模型+可训练草稿），小规模设备可能难以复现。
