## 摘要
LLM在生产环境中被广泛采用，将推理系统推向极限。分离式LLM服务（如PD分离和KV状态分离）提高了可扩展性和成本效率，但也将KV转变为跨网络和存储边界的显式负载，使KV成为端到端的主要瓶颈。现有的KV压缩通常是静态运行时配置，尽管生产服务上下文在工作负载组合、带宽和SLO/质量预算方面随时间变化。因此，固定选择可能不是最优的，甚至会增加延迟。我们提出了KVServe，这是首个面向服务感知且自适应的KV通信压缩框架，用于分离式LLM服务：KVServe（1）将KV压缩统一为模块化策略空间，包含新组件和跨方法重组；（2）引入了贝叶斯分析引擎，高效搜索该空间并提炼出3D帕累托候选集，将离线搜索开销降低50倍；（3）部署了服务感知的在线控制器，结合分析延迟模型和轻量级赌博机，在约束条件下选择配置文件并纠正离线到在线的不匹配。集成到vLLM中，在数据集、模型、GPU和网络上进行评估，KVServe在PD分离式服务中实现了高达9.13倍的JCT加速，在KV分离式服务中实现了高达32.8倍的TTFT减少。

## Abstract
LLMs are widely adopted in production, pushing inference systems to their limits. Disaggregated LLM serving (e.g., PD separation and KV state disaggregation) improves scalability and cost efficiency, but it also turns KV into an explicit payload crossing network and storage boundaries, making KV a dominant end-to-end bottleneck. Existing KV compression are typically static runtime configurations, despite production service context varies over time in workload mix, bandwidth, and SLO/quality budgets. As a result, a fixed choice can be suboptimal or even increase latency. We present \emph{KVServe}, the first service-aware and adaptive KV communication compression framework for disaggregated LLM serving: KVServe (1) unifies KV compression into a modular strategy space with new components and cross-method recomposition; (2) introduces Bayesian Profiling Engine that efficiently searches this space and distills a 3D Pareto candidate set, reducing $50\times$ offline search overhead; and (3) deploys a Service-Aware Online Controller that combines an analytical latency model with a lightweight bandit to select profiles under constraints and correct offline-to-online mismatch. Integrated into vLLM and evaluated across datasets, models, GPUs and networks, KVServe achieves up to $9.13\times$ JCT speedup in PD-separated serving and up to $32.8\times$ TTFT reduction in KV-disaggregated serving.

### 1. 核心问题与整体含义（研究动机和背景）

*   **背景与趋势**：大型语言模型（LLM）服务正从单机推理转向解耦架构，包括**预填充/解码（PD）分离**和**KV状态解耦**。这种架构虽提升了扩展性和成本效率，但将原本存在于GPU内存中的KV缓存变为了需要在网络和存储间传输的显式负载。
*   **核心问题**：随着上下文窗口增长，KV缓存体积巨大（如Llama 3.1-70B模型在128K tokens下产生39.06GB KV缓存），其传输成为端到端服务的**主要瓶颈**（图1显示KV通信时间可占任务完成时间的60%）。
*   **现有方案的不足**：现有的KV压缩方法（如CacheGen, KIVI）通常是**静态配置**的。然而，生产环境中的**服务上下文会动态变化**（如工作负载类型、有效带宽、服务质量（SLO）和质量预算）。一个固定的静态策略无法在所有情况下保持最优，甚至在部分情况下会导致**负优化**（即压缩和解压缩的开销超过了通信节省带来的收益，导致总延迟反而增加）。
*   **核心观点**：KV压缩不应是一个固定的算法选择，而是一个**依赖服务状态的、带约束的策略选择问题**。

### 2. 方法论：核心思想、关键技术

KVServe提出了一个三阶段的服务感知自适应KV压缩框架。

*   **核心思想**：将KV压缩视为一个受服务状态（工作负载、带宽、SLO、质量要求）约束的决策问题，通过离线搜索出帕累托最优策略集，并在运行时根据实时服务上下文自适应选择最佳策略。
*   **第一阶段：模块化策略池（Modular Strategy Pool）**
    *   **统一抽象**：将KV压缩抽象为三个可组合的模块化管道：**变换器 (Transformer, T)** -> **量化器 (Quantizer, Q)** -> **编解码器 (Codec, C)**。`BS = C(Q(T(X)))`。
    *   **组件实例化**：
        *   **T**: 包括ByPass, Delta, Hadamard, Affine等预处理方法。
        *   **Q**: 包括Naive, Various Dimension, Mixed Precision等量化方法。
        *   **C**: 包括ByPass, BitPacking, Entropy Coding等编码方法。
    *   **新组件 (MixHQ)**：提出混合精度头级量化（MixHQ），区别于硬性的丢弃，对不太重要的流式头（Streaming Heads）进行极低比特量化，而对重要的检索头（Retrieval Heads）保留高精度，实现更优的压缩率与准确率权衡。
*   **第二阶段：贝叶斯分析引擎（Bayesian Profiling Engine）**
    *   **目标**：高效地搜索指数级增长（超4000个候选）的策略空间，找出满足准确率约束下的最大压缩比候选。
    *   **关键技术**：
        1.  **高斯过程模型构建**：将策略配置映射到统一嵌入空间，用高斯过程代理模型来预测每个策略的压缩比和准确率。
        2.  **采集函数设计**：使用带约束的自定义采集函数（ACquisition Function, AF），平衡**开发**（选择满足准确率约束且期望压缩比高的配置）和**探索**（选择不确定性高的配置）。
        3.  **双向剪枝**：利用压缩比与准确率的单调关系进行双向剪枝。若当前候选不满足准确率约束，则剪掉所有压缩比更高的候选；若满足，则剪掉所有压缩比更低的候选。
        4.  **早停机制**：当连续多次失败或搜索空间耗尽时，提前终止搜索。
    *   **输出**：生成一个**3D帕累托前沿**（Acc-CR-Lat三维空间中的非支配点集），作为离线查找表供在线选择。
*   **第三阶段：服务感知在线控制器（Service-Aware Online Controller）**
    *   **目标**：在运行时感知服务上下文`c = (w, B, T_SLO, q_min)`，从帕累托前沿中快速选择延迟最小且满足SLO和质量约束的策略。
    *   **核心技术**：
        1.  **分析延迟模型**：构建分段JCT模型 `T_p(c) = T_model(w) + V/s_p + V/(B*cr_p)`，并通过推导得出**带宽阈值定理**：只有当有效带宽B低于某个阈值`B*_p`时，压缩策略`p`才是有益的。同时，最优策略在`x=1/B`上是分段常数。这提供了一个高效、可解释的基线选择机制。
        2.  **残差校正赌博机**：模拟运行时偏差（如GPU负载、队列竞争），通过轻量级**ε-贪心**算法在候选集中（模型最优策略+其相邻策略）进行探索，并使用指数加权移动平均（EWMA）在线更新这些策略的延迟预测残差，以修正离线与在线环境的漂移。
    *   **SLO安全护栏**：使用`T_p(c) <= T_SLO`作为保守可行性过滤器；对短期违反SLO的策略实行冷却机制。
